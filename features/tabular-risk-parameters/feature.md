# FEATURE-TABULAR-RISK-PARAMETERS — Табличные риск-параметры

Статус: **импортировано из legacy-плана и актуализировано по реальному backlog**  
Квартал: `2026-Q2`  
Дата обновления: `2026-05-04`

## Цель
Поднять в отдельную feature контур табличных риск-параметров: backend-модель и обработку, интеграцию с внешними источниками/запуском симуляции, а также frontend-форму изменения риск-параметров.

## Контекст
В legacy-планах этот контур долго жил вне основной feature-структуры как отдельный блок `Табличные риск-параметры (за рамками MVP)`, а в actualized tracking был материализован реальными backlog-задачами `RSCON-2429`, `RSCON-2430`, `RSCON-2431`, `RSCON-2432`. По новому обновлению добавляется и follow-up `RSCON-2452`, поэтому контур пора зафиксировать в актуальных `features/` и `planning/` артефактах.

## Ideal scope
- единая доменная модель табличных риск-параметров;
- backend-контур чтения, обработки и передачи табличных РП;
- интеграция с РП и ФП Симуляция без ручных временных склеек;
- frontend-форма изменения риск-параметров с устойчивой работой на длинных наборах данных;
- операционный контур временных файлов/очистки без накопления мусора.

## MVP scope
- backend API, БД и базовая логика табличных риск-параметров;
- получение и обработка табличных РП из смежного контура;
- передача табличных РП при запуске симуляции;
- scheduler очистки файлов как продолжение backend-контура;
- frontend-переделка формы изменения риск-параметров.

## Что исключено из MVP
- отдельная самостоятельная страница управления табличными РП;
- развитые аналитические представления сверх формы и интеграций;
- дополнительный QA-контур, не подтверждённый в текущем backlog;
- переописание уже существующего simulation UI вне этой дельты.
