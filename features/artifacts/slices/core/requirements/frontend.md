## Контекст и бизнес-смысл

### Цель

Пользователь должен видеть и, при наличии прав, редактировать список внешних документов, привязанных к доменной сущности.

### Пользователи и роли

- пользователи, имеющие право просмотра родительской сущности;
- пользователи, имеющие право редактирования родительской сущности.

## Границы MVP

### Входит в MVP

- отображение списка артефактов;
- add/edit/remove ссылок;
- валидация `title` и `url`;
- единая логика empty state.

### Не входит в MVP

- upload файлов;
- preview содержимого по ссылке;
- complex document workflow.

## Пользовательские сценарии

### Сценарий FE-1. Просмотр артефактов
1. Пользователь открывает host screen.
2. UI загружает список артефактов.
3. Пользователь открывает внешний документ по ссылке.

### Сценарий FE-2. Редактирование артефактов
1. Пользователь добавляет или меняет ссылку.
2. UI валидирует поля.
3. После успешного ответа список обновляется.

## UI-состав и навигация

### Экран/состояние 1. Блок артефактов

- Назначение: показать список ссылок и actions.
- Откуда открывается: из host screen сущности.
- Куда ведёт: во внешний документ или в inline edit flow.
- Что видно пользователю: title, type, description, link.
- Какие действия доступны: открыть, добавить, изменить, удалить.

## Функциональные требования

### FE-FR-1. Единый список артефактов

**Описание:**
Блок артефактов выглядит и работает одинаково на целевых host screens.

**Детали:**
- сортировка стабильна;
- empty state явно показан;
- ссылка открывается безопасно как внешний ресурс.

### FE-FR-2. Валидация ссылки

**Описание:**
Frontend не отправляет пустые или явно неверные данные.

**Детали:**
- `title` обязателен;
- `url` обязателен;
- невалидный URL блокирует сохранение.

## Валидация на frontend

### Правила

- нельзя сохранить артефакт без `title`;
- нельзя сохранить артефакт без валидного `url`.

### Сообщения об ошибках

| Ситуация | Сообщение | Где показываем |
|---|---|---|
| пустое название | `Укажите название артефакта` | форма блока |
| невалидная ссылка | `Укажите корректную ссылку` | форма блока |
