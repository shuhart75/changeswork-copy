## Что считается существующим поведением
- отдельная страница списка симуляций;
- таблица со столбцами `ID`, `Название`, `Режим`, `Статус`, `Этап`, `Дата обновления этапа`, `Автор`, `Дата создания`;
- фильтрация по продукту и статусу;
- поиск по названию;
- сортировка по столбцам;
- пагинация;
- переход в детальную страницу по клику на строку.

## UI-правила
- ПРМ видит симуляции по всем продуктам, но редактирование ограничено своим продуктом;
- методолог и админ видят все симуляции;
- статусные бейджи отражают simulation lifecycle, а не approval flow.

## Что важно для дальнейших дельт
- Этот экран является точкой входа в simulation scope, но не host screen для `simulation-bt-agent`.
- Новые изменения списка симуляций нужно трактовать как отдельную дельту поверх existing coverage.
