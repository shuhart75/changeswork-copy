## STORY-PACKAGES-001 — Страница пакетов

Slice card: `slices/page/slice.md`
Детализация FE: `slices/page/requirements/frontend.md`
Детализация BE: `slices/page/requirements/backend.md`
Прототип: `slices/page/delivery-prototype/prototype.html`
Planning story: `planning/stories/STORY-PACKAGES-001.md`

**Бизнес-требования**

- Цель: дать пользователю рабочее место для формирования пакетов и просмотра уже отправленных пакетов.
- Бизнес-правила: пакет собирается из элементов в `awaiting_ratification`; backend draft в MVP не используется; package detail остаётся read-only.
- Ограничения: расширенные вкладки history/in-progress/rejected и редактирование пакета вне scope.

**Пользовательские требования к АС КОДА**

- Пользователь видит очередь доступных элементов, может сформировать пакет и отправить его.
- Пользователь видит список `Мои пакеты` и может открыть read-only деталку.

**Критерии приемки**

1. Очередь элементов и локальный выбор работают по правилам MVP.
2. Созданный пакет отправляется через согласованный API и отображается в `Мои пакеты`.
3. Деталка пакета доступна только в режиме просмотра.

**USE CASES**

- **осн. сценарий 1** пользователь собирает пакет из очереди и отправляет его.
- **альт. сценарий 1.1** пользователь снимает часть элементов до отправки.
- **альт. сценарий 1.2** отправка завершается ошибкой и пользователь получает понятное сообщение.
- **осн. сценарий 2** пользователь открывает read-only деталку ранее отправленного пакета.
