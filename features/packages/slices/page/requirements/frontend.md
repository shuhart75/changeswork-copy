# Страница "Пакеты" (Frontend)

Статус: **исключено из MVP**  
Область: MVP  
Дата обновления: 2026-05-25  
Decision ID: `DEC-2026-05-25-APPROVALS-SBERDOCS-001`

## Решение

Frontend страницы `Пакеты` не реализуется. С переходом на SberDocs локальное пакетирование согласований в АС КОДА исключено для минимизации разработки.

## Superseded scope

Отменены:

- очередь элементов `awaiting_ratification`;
- выбор нескольких элементов в пакет;
- brief пакета;
- отправка пакета;
- список `Мои пакеты`;
- read-only деталка пакета;
- package cards на странице `Согласования`.

## Новое поведение

- Пользователь отправляет каждую целевую сущность в SberDocs из её host screen.
- Если пользователю нужен статус, он смотрит блок SberDocs status на host screen.
- Если пользователю нужно принять решение, он открывает SberDocs.

## Критерии приемки отмены

- [ ] В MVP нет маршрута `/packages` как рабочего экрана.
- [ ] UI не показывает package queue и package draft.
- [ ] UI не вызывает `POST /api/v1/packages`.
- [ ] Старый package prototype помечен как obsolete.
