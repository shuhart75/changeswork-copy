# Delivery Prototype Notes

Срез: `bt-publication`  
Дата обновления: `2026-04-27`

Покрытие прототипа:
- подходящий и неподходящий контекст для кнопки `Сформировать БТ`;
- tooltip-подсказки для доступного и недоступного состояния кнопки;
- локальная вставка черновика запроса по данным симуляции;
- свободное редактирование черновика пользователем;
- отсутствие автоотправки после вставки;
- показ `btUrl` после успешной публикации и ручное копирование ссылки пользователем.

Что важно для handoff:
- черновик строится на фронте из имени пользователя, риск-параметров и бизнес-цели;
- под полем ввода остаются только две основные кнопки: `Сформировать БТ` и `Отправить`;
- при неподходящем контексте окно агента не исчезает, меняется только доступность действия по БТ;
- `btUrl` показывается в окне агента;
- автоматического обновления поля `Ссылка на БТ` на host page в MVP нет;
- пользователь копирует ссылку из окна агента вручную и при желании сам сохраняет её в соответствующем разделе симуляции.
