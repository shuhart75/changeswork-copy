# Delivery Prototype Notes

Срез: `dialog-session`  
Дата обновления: `2026-04-27`

Покрытие прототипа:
- правая неблокирующая панель диалога;
- восстановление истории по `session_id`;
- раздельные действия открытия окна и отправки сообщения;
- передача `user_prompt`, ФИО из СУДИР и `dialog_started_at` при отправке;
- отображение сообщений с поддержкой Markdown;
- status chip агента в шапке, сообщение об ошибке и рекомендация перезапустить сессию.

Что важно для handoff:
- при состоянии `loading` или `error` основной интерфейс не блокируется;
- поле ввода и кнопка отправки блокируются только локально внутри панели;
- поле ввода имеет фиксированную высоту в 5 строк и внутренний вертикальный scroll;
- между шапкой панели и чатом нет промежуточных информационных карточек;
- в макете показан обмен текстом между интерфейсом АС КОДА и окном агента в обе стороны.
