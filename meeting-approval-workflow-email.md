# Итоги встречи по процессу согласования изменений риск-стратегий в АС КОДА

**Тема письма:** Итоги обсуждения процесса согласования изменений риск-стратегий в АС КОДА

Коллеги, добрый день.

По итогам встречи 15 мая по процессу согласования изменений риск-стратегий предлагаю зафиксировать основные выводы и дальнейшие шаги.

Ключевой вывод встречи: целевой процесс требует дополнительного управленческого решения по роли АС КОДА и документарного контура СберДокс/SberBox. Сейчас важно не только реализовать удобный workflow согласования, но и корректно закрыть вопросы юридической значимости, хранения эталонных документов, конфиденциальности и ответственности.

Предлагаемый целевой подход:

- **АС КОДА** используется как единое окно управления изменением: создание изменения, маршрут согласования, статусы, аудит, связь с внедрением, кодом, логами и актуальной версией стратегии.
- **СберДокс/SberBox** сохраняется как юридически значимый документарный контур: хранение эталонных документов, подписи, делегирование полномочий и работа с материалами повышенной конфиденциальности.

Такой вариант позволяет реализовать управляемый процесс в АС КОДА, не дублируя чувствительные документы и не создавая заново юридический механизм подписи и делегирования внутри АС КОДА.

Отдельно необходимо подтвердить следующие вопросы:

1. Является ли действие в АС КОДА юридически значимым утверждением или операционным подтверждением в рамках workflow.
2. Где хранится эталонная версия стратегии и изменений к ней.
3. Какие данные можно хранить непосредственно в АС КОДА, а какие должны храниться только в документарном контуре со ссылкой из АС КОДА.
4. Какой уровень конфиденциальности применяется к дельте изменения, описанию стратегии, коду и связанным материалам.
5. Требуется ли СберДокс/SberBox для подписи распоряжения и фиксации утверждения.
6. Как процесс будет описан для ПСИ: через временное распоряжение или через ARIS.

С учетом этих открытых вопросов предлагается организовать отдельную встречу с Джангиром для подтверждения целевой модели. На встречу целесообразно выходить не с перечнем проблем, а с вариантами решения и рекомендуемой гибридной моделью:

- АС КОДА - единое окно и workflow согласования;
- СберДокс/SberBox - юридически значимый контур документов, подписей и эталонных версий.

После подтверждения целевой модели предлагаются следующие действия:

| Действие | Ожидаемый результат |
|---|---|
| Зафиксировать целевую модель процесса с Джангиром | Подтверждено разделение ролей АС КОДА и СберДокс/SberBox |
| Подготовить драфт процесса для ПСИ | Описан временный процесс или основа для ARIS |
| Определить уровни конфиденциальности | Зафиксирована матрица хранения данных, документов и ссылок |
| Доработать требования к АС КОДА | Учтены статусы, аудит, ссылки на документы, права доступа и интеграция со СберДокс |
| Провести аналитику и оценить новые объемы разработки | Подготовлена обновленная оценка трудоемкости и влияния на текущий план |
| Согласовать сдвиг сроков разработки и поставки | Зафиксированы обновленные сроки с учетом новой аналитики и подтвержденного объема работ |
| Подготовить демонстрационный сценарий | Показано единое окно согласования в АС КОДА с корректной связью с документарным контуром |

Прошу подтвердить, что такой подход можно использовать как основу для дальнейшей проработки, встречи с Джангиром и последующего согласования влияния на сроки разработки и поставки.
