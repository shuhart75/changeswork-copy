# QA Phase 2 — новые списки и формы

Дата обновления: `2026-04-24`
Источник: `context/source-materials/legacy/changeswork-full/planning/mvp/current/tasks/legacy/mvp_tasks_qa_phase2.md`
Оценка legacy: `10 дней`

## Фокус фазы

Проверка нового пользовательского контура вокруг списков и форм:
- список `Changes`;
- список `Scorecards` с Excel-экспортом;
- форма `Pilot`;
- интеграционные связки между `Simulation`, `Scorecard`, `Pilot`, `Deployment`.

## Ключевые проверки

### Lists
- отображение данных, пагинация, сортировка, фильтрация;
- поиск по ID и названию;
- продуктовые ограничения доступа;
- переходы на detail screens.

### Scorecards export
- экспорт текущей страницы и полного набора с учётом фильтров;
- корректный `.xlsx`-файл;
- корректность содержимого и имени файла.

### Pilot form
- обязательность хотя бы одной скоркарты;
- multi-select с поиском и chips;
- фильтрация скоркарт по продукту внедрения;
- создание и редирект на деталку;
- error handling.

### Интеграционные сценарии
- создание сущностей и проверка связей между ними;
- если `Simulation` уже есть в целевой системе, проверка цепочки `Simulation -> Scorecard -> Pilot/Deployment`.

## Acceptance frame

Минимум, который legacy считал обязательным:
- корректная работа списков и фильтров;
- работоспособный Excel export;
- валидная форма пилота;
- корректное создание связей между сущностями;
- корректные права доступа;
- задокументированные баги и исправление критичных дефектов.
