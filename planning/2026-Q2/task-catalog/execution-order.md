# Execution order — 2026-Q2 MVP

Дата обновления: `2026-04-24`
Источник: `planning/2026-Q2/imported-source/tasks/mvp_tasks_execution_order.md`

## Роль документа

Кратко фиксирует рекомендуемый порядок запуска потоков внутри MVP без углубления в каждую задачу.

## Последовательность верхнего уровня
1. `Roles`
2. `Artifacts`
3. `Approvals / ratification` и параллельная ветка `Packages`
4. `Deployments`
5. `Scorecards`
6. `Notifications`
7. `Trace`
8. `QA`

## Практический смысл
- этот порядок нужен не как канонический baseline, а как historical planning intent;
- он помогает понять, почему некоторые задачи в gantt были привязаны друг к другу;
- для детального содержимого каждого потока нужно переходить в `source-alignment.md` и feature-level material.
