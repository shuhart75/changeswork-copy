## Бизнес-требования

### User Story

Я как методолог / ПРМ (EDITOR / METODOLOG) АС КОДА хочу заполнять бриф в составе вкладки «Цепочка согласования», чтобы:
- подготавливать документ для утверждающего в рамках одного пользовательского сценария
- не вводить вручную данные, уже известные системе из маршрута и скоркарты
- видеть итоговое представление брифа в предпросмотре до его окончательного сохранения

### Краткое бизнес-описание

- Необходимо реализовать блок брифа в составе вкладки «Цепочка согласования» карточки сущности
- Бриф предназначен для подготовки документа, передаваемого утверждающему
- Бриф не является отдельной бизнес-сущностью со своей статусной моделью. Его жизненный цикл полностью определяется жизненным циклом активной цепочки согласования

### Область действия требований

Требования распространяются на блок брифа вкладки «Цепочка согласования»:
- состав и поведение полей брифа
- правила предзаполнения системных полей
- правила валидации при сохранении
- блок скоркарты, встроенный в вкладку
- модальное окно предпросмотра

### Ключевые принципы архитектуры

1. Бриф является частью вкладки «Цепочка согласования» и не образует отдельной статусной модели
2. Доступность брифа для редактирования определяется статусом активной цепочки согласования
3. Данные брифа хранятся в составе JSON-структуры активной версии цепочки и сохраняются вместе с ней одним методом
4. При открытии вкладки пользовательские поля заполняются из ранее сохранённого JSON, системные — из актуальных данных системы
5. Скоркарта отображается на вкладке только для просмотра
6. Предпросмотр является агрегированным представлением брифа только для чтения

## Критерии приемки

### Отображение брифа при наличии активной цепочки

- 61: Блок брифа отображается на вкладке при наличии активной цепочки
- 62: Пользовательские поля заполнены значениями из JSON активной версии цепочки
- 63: Поле «Эффекты (влияние)» заполнено данными из активной скоркарты и недоступно для ввода
- 64: Блок согласующих сформирован из текущего маршрута
- 65: Блок скоркарты отображается в режиме read-only

### Отсутствие активной цепочки

- 66: Бриф не отображается как самостоятельный редактируемый объект
- 67: Кнопка «Редактировать» недоступна или отсутствует

### Переход в режим редактирования

- 68: Страница переходит в режим редактирования одним нажатием кнопки «Редактировать»
- 69: Ранее отображавшиеся данные сохраняются и остаются видимыми
- 70: Пользовательские поля брифа доступны для ввода
- 71: Поле «Эффекты (влияние)» остаётся недоступным для ручного ввода
- 72: Блок скоркарты остаётся в режиме read-only

### Сохранение брифа совместно с цепочкой

- 73: Данные брифа сохраняются совместно с новой версией цепочки одним запросом
- 74: Предыдущая версия цепочки деактивируется
- 75: Страница переходит в режим просмотра с актуальными данными
- 76: Данные брифа предыдущей версии цепочки остаются привязанными к ней в истории

### Предпросмотр

- 77: Предпросмотр открывается в модальном окне
- 78: Все поля отображаются в режиме read-only
- 79: Раздел «Основные правила / изменения» отображается как простой список
- 80: Данные скоркарты присутствуют в preview
- 81: Элементы управления формой в preview отсутствуют
- 82: Повторное открытие preview не приводит к задвоению содержимого

### Автозаполнение поля «Эффекты (влияние)»

- 83: Поле содержит значение, сформированное только из метрик, не исключённых из расчёта
- 84: Если все метрики исключены, поле отображается пустым
- 85: Поле недоступно для ручного редактирования

### Раздел «Основные правила / изменения»

- 86: При добавлении элемента типа «Новое правило» отображается одно текстовое поле
- 87: При добавлении элемента типа «Изменение» отображаются поля AS IS и TO BE
- 88: Пользователь удаляет отдельный элемент без потери остальных
- 89: Порядок элементов сохраняется при сохранении и отображении

### Валидация при сохранении

- 90: При незаполненных обязательных полях сохранение не выполняется
- 91: Незаполненные обязательные поля подсвечиваются
- 92: Рядом с каждым незаполненным полем отображается конкретное сообщение об ошибке
- 93: Пользователь остаётся в режиме редактирования после неуспешной попытки сохранить
- 94: Открытие предпросмотра доступно при незаполненном брифе

### Блокировка редактирования по статусу цепочки

- 95: При статусе цепочки, не допускающем редактирование, бриф отображается в режиме просмотра
- 96: Кнопка «Редактировать» недоступна или скрыта
- 97: Поля брифа недоступны для ввода

### Версионирование данных брифа

- 98: Данные брифа новой версии цепочки соответствуют значениям, сохранённым в момент создания этой версии
- 99: Данные брифа предыдущей версии соответствуют значениям, существовавшим до редактирования
- 100: Каждая версия цепочки содержит независимый набор данных брифа

## Требования Frontend

### 1. Страница цепочки согласования

Страница поддерживает два режима:
- режим просмотра
- режим редактирования

### 2. Кнопка «Редактировать»

Единая кнопка «Редактировать» переводит всю страницу в режим редактирования без отдельного экрана.

### 3. Отображение брифа

Бриф отображается как вспомогательный блок данных. Доступность брифа определяется:
- наличием активной цепочки
- статусом активной цепочки
- правами пользователя на редактирование
- общим режимом страницы

### 4. Экраны и элементы брифа

| Элемент | Действие / поведение |
|---|---|
| Заголовок блока брифа | Обозначает раздел брифа |
| Поле адресата | Позволяет вводить или отображать обращение |
| Поле реквизитов / номера документа | Позволяет вводить или отображать реквизиты |
| Поле наименования документа | Позволяет вводить или отображать название |
| Блок согласующих | Отображает список согласующих из маршрута |
| Поле «Цель» | Позволяет вводить или отображать цель |
| Поля периметра применения | Позволяют вводить или отображать параметры |
| Раздел «Основные правила / изменения» | Редактирование списка правил и изменений |
| Поле «Эффекты (влияние)» | Отображает системно сформированное значение, недоступно для ручного ввода |
| Поле «Риски» | Позволяет вводить или отображать риски |
| Поле «Заключение Антифрод» | Позволяет вводить или отображать заключение |
| Кнопка «Предпросмотр» | Открывает модальное окно итогового preview |

### 5. Правила доступности брифа на UI

- если активная цепочка отсутствует — бриф отображается как информационный блок без возможности редактирования
- если статус цепочки допускает редактирование — бриф редактируется в составе общей страницы
- если статус цепочки не допускает редактирование — бриф только для просмотра
- для статуса `in_ratification` редактирование брифа недоступно

### 6. Предзаполнение данных на frontend

- Frontend использует уже существующие данные предзаполнения
- Значение поля «Эффекты (влияние)» подставляется из данных активной скоркарты
- Frontend не реализует отдельную бизнес-логику расчёта эффектов

### 7. Раздел «Основные правила / изменения»

Раздел поддерживает два типа элементов:
- новое правило (одно текстовое поле)
- изменение (поля AS IS и TO BE)

**В режиме preview** раздел отображается простым списком.

### 8. Валидация полей брифа

**Обязательные поля**:
| Поле | Обязательность | Примечание |
|---|---|---|
| Адресат | Обязательное | Текстовое поле |
| Наименование документа | Обязательное | Текстовое поле |
| Цель | Обязательное | Текстовое поле |
| Реквизиты / номер документа | Уточнить | — |
| Периметр применения | Уточнить | — |
| Риски | Необязательное | — |
| Заключение Антифрод | Необязательное | — |

### 9. Скоркарта в составе страницы

Скоркарта отображается как read-only блок:
- заголовок скоркарты
- критичность
- название
- финансовые метрики и их значения
- признак исключения метрик
- индикаторы

### 10. Предпросмотр

- открывается в модальном окне
- отображает данные в read-only виде
- не содержит элементов редактирования формы
- раздел «Основные правила / изменения» как простой список

### 11. Нефункциональные требования frontend

- Переход в режим редактирования не приводит к потере данных
- Повторное открытие preview не приводит к задвоению контента
- Экран устойчив к частично пустым данным
- Frontend не вводит отдельную статусную модель брифа

## Требования Backend

### Общие принципы

Backend предоставляет данные для определения состояния страницы, получения и сохранения JSON-данных брифа в составе активной цепочки.

### Модель хранения брифа

Бриф хранится как JSON-данные, относящиеся к активной версии цепочки согласования. Отдельная статусная модель не вводится.

### Логический состав данных брифа

| Группа | Содержание |
|---|---|
| Реквизиты | адресат, реквизиты документа, наименование документа |
| Содержательная часть | цель, периметр применения |
| Изменения | список структурированных элементов |
| Системные данные | согласующие, эффекты |
| Дополнительные поля | риски, заключение Антифрод |

### Поле «Эффекты»

- Поле «Эффекты» заполняется из данных активной скоркарты
- Backend игнорирует произвольное пользовательское значение для поля «Эффекты (влияние)»

### Серверная валидация данных брифа

- Backend возвращает HTTP-статус 422 с описанием ошибок при валидации
- Backend отклоняет сохранение, если присутствует пользовательское значение поля «Эффекты (влияние)»
- Backend отклоняет сохранение, если версия цепочки не является актуальной

### Матрица ответственности

| Область | Frontend | Backend |
|---|---|---|
| Доступность брифа | Отображает по статусу цепочки | Возвращает статус активной цепочки |
| Единый режим редактирования | Переключает режим страницы | Возвращает допустимость изменения |
| Хранение брифа | Работает с данными формы | Хранит JSON-данные брифа в составе цепочки |
| Автоподстановка согласующих | Отображает | Отдаёт данные маршрута |
| Получение активной скоркарты | Использует существующий контракт | Предоставляет существующий контракт |
| Подстановка эффектов | Берёт значение из данных скоркарты | Отдаёт эффекты в составе данных скоркарты |
| Read-only скоркарта | Обеспечивает | Возвращает данные просмотра |
| Версионирование | Показывает актуальную версию | Сохраняет новую версию цепочки вместе с брифом |
| Валидация брифа | Клиентская валидация обязательных полей | Серверная валидация, возврат ошибок по полям |

## Граничные случаи

| Ситуация | Ожидаемое поведение |
|---|---|
| Активная цепочка отсутствует | Бриф отображается в информационном режиме |
| Цепочка в статусе `in_ratification` | Бриф редактируется в рамках общей страницы |
| Создана новая версия цепочки | Отображаются данные брифа новой версии |
| Предыдущая версия цепочки архивирована | Её бриф остаётся частью исторических данных |
| Все метрики исключены в скоркарте | Поле «Эффекты» отображается пустым |
| Эффекты в ответе скоркарты | Поле «Эффекты (влияние)» заполняется без дополнительного расчёта |
| В разделе изменений нет элементов | Preview отображает пустой раздел или скрывает его |
| Конкурентное редактирование | Backend возвращает ошибку конфликта версий |
| Потеря соединения при сохранении | Frontend отображает сообщение об ошибке; данные сохраняются в памяти |
