## Содержание

### Введение

Согласование релизов риск-параметров с интеграцией в SberDocs для автоматизации процесса согласования.

### Функциональные требования

#### UI.01 Форма отправки релиза на согласование

##### UI.01.01 Экран "Форма отправки релиза на согласование"

| Элемент UI | Label | Содержимое | Действие |
|------------|-------|------------|----------|
| Поле ввода | Номер релиза | Текущий номер релиза, readonly | - |
| Поле ввода | Название релиза | Из поля title релиза, readonly | - |
| Поле ввода | Описание релиза | Из поля description релиза | - |
| Textarea | Краткое содержание документа | О корректировке значений, расположенных в ФП "Риск-параметры" и иных корректировках риск-стратегий | Дать отредактировать текст поля |
| Список | Приложенные документы | Список приложенных документов | Напротив каждого документа кнопка удаления |
| Кнопка | - | Отмена | Очистить маршрут, очистить список документов, закрыть окно |
| Кнопка | - | Согласование не требуется | Показать confirm с обоснованием, затем очистить маршрут и вывести релиз в пром |
| Кнопка | - | Приложения | Открыть окно выбора множества документов с локального компьютера |
| Кнопка | - | Выбрать маршрут согласования | Закрыть UI.01.01 и открыть UI.01.02 |

##### UI.01.02 Экран "Маршрут согласования"

| Элемент UI | Label | Содержимое | Действие |
|------------|-------|------------|----------|
| Список/Таблица | Выберите маршрут согласования | ФИО сотрудников с ролями: авторы (1-3), согласующие, получатели, подписанты | Добавить/удалить сотрудника |
| Поле ввода + Кнопка "Добавить" | Табельный номер сотрудника | Пусто | Добавить сотрудника в маршрут |
| Выпадающий список | Подразделение регистрации | ЦА-428, ЦА-376, ЦА-1 | Выбор подразделения |
| Кнопка | - | Назад | Закрыть UI.01.02 и открыть UI.01.01 |

| Кнопка | - | Согласование не требуется | Аналогично экрану 1 |
| Кнопка | - | Отправить на согласование | Проверить все 4 роли, отправить запрос в бек |

#### UI.02 Доработка отображения номера документа SberDocs

Добавить столбец "SberDocs" на страницах:
- "Параметры риск-стратегий"
- "На утверждении"
- "Доступны для отката"

Выводить `systemNumber` при наличии, иначе оставить пустым.

#### UI.03 Доработка формы "На утверждение"

Добавить кнопку "Согласование". Поддержать новые статусы:

| Код статуса | Текст на эф | Комментарий |
|-------------|-------------|-------------|
| PROCESS_OF_APPROVAL | Согласование | Скрыть кнопки меню |
| APPROVED | Согласован | - |

### Интеграция со SberDocs (INT.01)

#### INT.01.01 Создание задания на создание документа

**POST**: `/public/Gateway/document-job`

##### Запрос в SberDocs

```json
{
  "externalDocumentId": "releaseId",
  "type": "INTERNAL",
  "kind": "DIVISION_ORDER",
  "senderList": [...],
  "recipientList": [...],
  "additionalAuthorList": [...],
  "author": {...},
  "summary": "О корректировке значений...",
  "documentFile": {...},
  "route": {...},
  "attachmentList": [...],
  "hasSignatureDetached": false,
  "hasRmsProtected": false,
  "externalDocumentSource": "PPRB_CM_FL",
  "registration": [...]
}
```

##### Ответ от SberDocs (http 200)

```json
{
  "externalDocumentId": "№2056/кр-22",
  "jobId": "c53336ba-b079-478a-b81e-1746753450ec",
  "jobState": "IN_WORK",
  "descriptionMessage": "...",
  "documentId": "...",
  "systemNumber": "Сн-ЦА/1-21",
  "tenant": "SBERBANK"
}
```

#### INT.01.02 Проверка статуса создания документа

**GET**: `/public/Gateway/document-job/{jobId}/state`

Получить `documentId` и `systemNumber` при статусе `COMPLETED`.

#### INT.01.03 Проверка статуса документа

**GET**: `/public/Gateway/document/{documentId}/state`

Если `documentState` в (APPROVED, EXECUTED) — разрешить вывод в пром.
