# Предложение по оптимизации работы через специализированных агентов

**Дата:** 2026-03-11
**Версия:** 1.0

---

## Анализ текущей ситуации

### Метрики проекта
- **70+ markdown файлов** документации
- **206.5 дней** последовательной работы
- **56 дней** критический путь при оптимальной параллелизации
- **24 задачи** в MVP (страницы, формы, процессы)
- **Архитектура:** Initiative-центричная с 9 ограниченными контекстами

### Выявленные паттерны работы

1. **Повторяющиеся структуры:**
   - 7 list views (списки сущностей)
   - 5 detail views (детальные страницы)
   - 5 форм создания/редактирования
   - Все следуют единому паттерну Material-UI + React

2. **Критические задачи (из TODO.md):**
   - Валидация форм (высокий приоритет)
   - Разделение ролей Approver/Ratifier (критический приоритет)
   - Процесс согласования (8 дней, критический путь)

3. **Сложные области:**
   - Approval workflow (двухэтапный процесс)
   - Lineage visualization (цепочки связей)
   - Множественные роли пользователей
   - Синхронизация терминологии (70+ файлов)

## Рекомендуемые специализированные агенты

### 🔴 Критический приоритет

#### 1. **approval-workflow-specialist**
**Назначение:** Специалист по процессу согласования/утверждения

**Зона ответственности:**
- Реализация двухэтапного процесса (Approval → Ratification)
- Разделение ролей Approver/Ratifier
- Логика назначения согласующих
- Валидация прав доступа на этапах
- Интеграция с жизненными циклами Pilot/Deployment

**Почему критично:**
- На критическом пути (8 дней backend)
- Блокирует страницы Packages и Approvals (17 и 15 дней)
- Сложная бизнес-логика с множественными ролями
- Высокий риск ошибок в правах доступа

**Экономия времени:** 30-40% (с 8 дней до 5-6 дней)

**Триггеры использования:**
- "реализуй процесс согласования"
- "добавь этап approval/ratification"
- "настрой права для approver/ratifier"
- "интегрируй workflow с жизненным циклом"

---

#### 2. **form-validation-specialist**
**Назначение:** Специалист по валидации форм

**Зона ответственности:**
- Client-side валидация (React Hook Form + Yup/Zod)
- Server-side валидация (Django REST Framework serializers)
- Валидация обязательных полей
- Валидация связей (минимум 1 скоркарта, и т.д.)
- Генерация сообщений об ошибках (русский язык)

**Почему критично:**
- В TODO.md как высокий приоритет
- 5 форм требуют валидации
- Единообразие валидации критично для UX
- Предотвращение некорректных данных

**Экономия времени:** 25-35% на формах (с 34 дней до 23-26 дней)

**Триггеры использования:**
- "добавь валидацию в форму"
- "проверь обязательные поля"
- "валидируй минимум N скоркарт"
- "настрой client-side/server-side валидацию"

---

#### 3. **crud-pattern-generator**
**Назначение:** Генератор CRUD-страниц по паттернам

**Зона ответственности:**
- Генерация list views (таблицы с фильтрами, сортировкой, пагинацией)
- Генерация detail views (вкладки, inline формы, связанные сущности)
- Генерация форм создания/редактирования
- Применение единого стиля Material-UI
- Соблюдение паттерна `width: '100%'` для корневых Box

**Почему критично:**
- 17 похожих страниц (списки + детальные + формы)
- Высокая степень повторяемости кода
- Риск несоответствия паттернам

**Экономия времени:** 40-50% на UI (с 77 дней frontend до 40-45 дней)

**Триггеры использования:**
- "создай list view для [сущность]"
- "создай detail view для [сущность]"
- "создай форму для [сущность]"
- "добавь вкладку с [контент]"

---

### 🟡 Высокий приоритет

#### 4. **domain-consistency-validator**
**Назначение:** Валидатор согласованности доменной модели

**Зона ответственности:**
- Проверка соответствия domain_model ↔ data_model ↔ state_machine
- Валидация бизнес-правил в коде
- Проверка терминологии (расширение project-consistency-manager)
- Валидация связей между сущностями

**Почему важно:**
- Уже есть project-consistency-manager (можно расширить)
- 70+ файлов требуют синхронизации
- Критично для целостности системы

**Экономия времени:** 20-30% на аналитике и проверках

**Триггеры использования:**
- "проверь согласованность модели"
- "валидируй бизнес-правила"
- "найди несоответствия между domain и data model"
- "проверь все связи [сущность]"

---

#### 5. **state-machine-validator**
**Назначение:** Валидатор машин состояний

**Зона ответственности:**
- Проверка соответствия PlantUML диаграмм и кода
- Валидация переходов состояний
- Проверка доступных действий в каждом состоянии
- Генерация тестов для state transitions

**Почему важно:**
- 5 сущностей с жизненными циклами
- Сложные переходы с условиями
- Критично для корректности бизнес-логики

**Экономия времени:** 15-25% на lifecycle задачах

**Триггеры использования:**
- "проверь state machine для [сущность]"
- "валидируй переходы состояний"
- "сгенерируй тесты для lifecycle"
- "обнови PlantUML диаграмму"

## Оценка общего эффекта

### Без специализированных агентов
- Критический путь: **56 дней**
- Общий объем: **206.5 дней**
- Риск ошибок: **Высокий**

### Со специализированными агентами (Фаза 1-2)
- Критический путь: **38-42 дня** (сокращение на 25-32%)
- Общий объем: **140-160 дней** (сокращение на 22-32%)
- Риск ошибок: **Средний-Низкий**

### Дополнительные преимущества
- ✅ Единообразие кода (паттерны применяются автоматически)
- ✅ Снижение cognitive load (агент знает контекст)
- ✅ Меньше ошибок в сложных областях (approval workflow)
- ✅ Быстрая адаптация к изменениям (агент обновляет связанные части)

## Рекомендации по реализации

### 1. Начать с approval-workflow-specialist
- Самый критичный блокер
- Сложная бизнес-логика
- Высокий риск ошибок

### 2. Параллельно создать form-validation-specialist
- Независим от approval workflow
- Закрывает TODO высокого приоритета
- Применим ко всем формам

### 3. Добавить crud-pattern-generator после первых двух
- Требует стабильных паттернов
- Максимальная экономия на UI
- Можно применить к оставшимся 10+ страницам

### 4. Расширить project-consistency-manager
- Уже существует как skill
- Добавить domain-consistency-validator функционал
- Интегрировать state-machine-validator

## Заключение

Внедрение 3 критических специализированных агентов может сократить время разработки MVP на **25-32%** (с 56 до 38-42 дней на критическом пути) при одновременном повышении качества и снижении количества ошибок.

Рекомендуется начать с **approval-workflow-specialist** и **form-validation-specialist** как наиболее критичных для проекта.
