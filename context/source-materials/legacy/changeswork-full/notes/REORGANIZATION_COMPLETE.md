### 2. Создан skill spec-sync

**Расположение:** `.claude/skills/spec-sync/`

**Назначение:** Автоматическая проверка синхронизации между документами спецификации

**Использование:**
```bash
# Проверка несоответствий
python3 .claude/skills/spec-sync/check_sync.py
```

**Что проверяет:**
- Наличие всех сущностей во всех документах
- Соответствие связей между domain_model.md и data_model.puml
- Соответствие статусов между state_machine.puml и domain_model.md

**Результат первой проверки:**

- ✓ Критичных ошибок нет

## 💡 Как работать с новой структурой

### При изменении спецификации:

1. Изменить файл в spec/
2. Запустить проверку синхронизации:
   ```bash
   python3 .claude/skills/spec-sync/check_sync.py
   ```
3. Исправить несоответствия в других файлах
4. Закоммитить изменения

### При создании новой версии:

1. Скопировать текущие файлы в spec/versions/vX.Y/
2. Обновить файлы в spec/
3. Обновить README.md (версия)

### При работе с прототипом:

1. Изменить prototypes/current.html
2. Проверить валидность:
   ```bash
   python3 scripts/validate_prototype.py prototypes/current.html
   ```
3. Открыть в браузере и протестировать
