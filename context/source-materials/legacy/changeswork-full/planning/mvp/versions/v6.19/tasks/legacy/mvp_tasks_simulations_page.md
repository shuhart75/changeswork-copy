# Задачи для страницы "Симуляции" (Simulations List View)

## Обзор страницы

**Страница:** Список симуляций (Simulations List View)
**Путь в макете:** Боковое меню → "Симуляции"
**Статус:** Вне скоупа MVP (уже реализовано, не меняем)

> Примечание по скоупу MVP: симуляции считаем уже реализованными (страницы/API/ЖЦ). В рамках MVP v6.5+ мы симуляции не дорабатываем; этот файл оставлен как справка.

### Что показывает страница (из макета):

**Табличное представление со столбцами:**
- ID (SIM-XXX)
- Название симуляции
- Режим симуляции (Стандартный/A/B тестирование/и т.д.)
- Статус (draft/running/completed/failed/archived)
- Этап (текущий этап выполнения)
- Дата обновления этапа
- Автор
- Дата и время создания

**Функционал:**
- Фильтрация по продукту (dropdown)
- Фильтрация по статусу
- Поиск по названию
- Сортировка по столбцам
- Пагинация
- Клик на строку → переход к детальному просмотру

**Права доступа:**
- ПРМ видит симуляции по всем продуктам (read-only для чужих), но создавать/редактировать может только симуляции своего продукта
- Методолог видит все симуляции
- Админ видит все симуляции

## BE-S1: Backend для страницы "Симуляции"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 2 дня

### Summary
Адаптация API endpoint для списка симуляций с фильтрацией, пагинацией и правами доступа

### Description

Адаптировать Backend API для страницы списка симуляций:

1. **API Endpoint: GET /api/simulations/**

   **Параметры запроса:**
   - `product_id` (optional) - фильтр по продукту
   - `status` (optional) - фильтр по статусу (draft/running/completed/failed/archived)
   - `search` (optional) - поиск по названию (case-insensitive, partial match)
   - `page` (optional, default=1) - номер страницы
   - `page_size` (optional, default=20) - размер страницы
   - `ordering` (optional, default='-created_at') - сортировка

   **Формат ответа:**
   ```json
   {
     "count": 42,
     "next": "http://api/simulations/?page=2",
     "previous": null,
     "results": [
       {
         "id": "uuid",
         "display_id": "SIM-001",
         "name": "Симуляция оптимизации лимитов",
         "mode": "standard",
         "status": "running",
         "stage": "Подготовка данных",
         "stage_updated_at": "2026-03-05T14:30:00Z",
         "created_at": "2026-03-01T10:00:00Z",
         "created_by": {
           "id": "user-uuid",
           "full_name": "Иван Иванов"
         }
       }
     ]
   }
   ```

2. **Бизнес-логика:**
   - Фильтрация по правам доступа:
     - ПРМ видит все (read-only для чужих)
     - Методолог видит все
     - Админ видит все
   - Поиск по названию (case-insensitive, ILIKE для PostgreSQL)
   - Сортировка по умолчанию: -created_at (новые первые)

3. **Права доступа:**
   - Middleware проверяет роль пользователя
   - Для list/read endpoints ПРМ может читать все продукты; ограничения применяются к операциям изменения (create/update/delete/submit)

4. **Оптимизация:**
   - Select related для created_by
   - Индексы на product_id, status, created_at

### Acceptance Criteria

- [ ] GET /api/simulations/ endpoint адаптирован
- [ ] Фильтрация по product_id, status работает
- [ ] Поиск по названию работает (case-insensitive)
- [ ] Пагинация работает корректно (default page_size=20)
- [ ] Сортировка по умолчанию: -created_at
- [ ] Права доступа реализованы (ПРМ читает все продукты; изменения только для своего продукта)
- [ ] Select related оптимизация применена
- [ ] Unit тесты покрывают все сценарии (coverage > 80%)
- [ ] API документация обновлена

### Dependencies
- AN-S1 (требуется API контракт)

### Technical Notes

**Пример SQL оптимизации (Django ORM):**
```python
from django.db.models import Q

Simulation.objects.filter(
    Q(name__icontains=search),
    product_id=product_id,
    status=status
).select_related(
    'created_by'
).order_by('-created_at')
```

## FE-S1: Frontend для страницы "Симуляции"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 4 дня

### Summary
Адаптация UI страницы списка симуляций с таблицей, фильтрами и поиском

### Description

Адаптировать Frontend для страницы списка симуляций:

1. **Компонент SimulationsView:**
   - Табличное представление (Material-UI Table)
   - Столбцы: ID, Название, Режим, Статус, Этап, Дата обновления этапа, Автор, Дата создания
   - Responsive дизайн

2. **Фильтры и поиск:**
   - Dropdown фильтр по продукту
   - Dropdown фильтр по статусу
   - Поле поиска по названию (debounce 300ms)
   - Кнопка "Сбросить фильтры"

3. **Сортировка:**
   - По умолчанию: по дате создания (новые первые)
   - Клик на заголовок столбца → сортировка
   - Индикатор направления сортировки

4. **Пагинация:**
   - Material-UI Pagination компонент
   - Показывать "Показано X-Y из Z"
   - Выбор размера страницы (10/20/50)

5. **Статусные бейджи:**
   - draft: серый
   - running: синий (анимация)
   - completed: зеленый
   - failed: красный
   - archived: серый с opacity

6. **Навигация:**
   - Клик на строку → переход к детальному просмотру
   - Breadcrumbs: "Симуляции"

7. **Loading и Error состояния:**
   - Skeleton loader при загрузке
   - Error message при ошибке API
   - Empty state если нет данных

8. **API интеграция:**
   - SimulationService.getList(filters, page, pageSize, ordering)
   - Обработка ошибок (403, 404, 500)
   - Toast уведомления при ошибках
