# Задачи для страницы "Пилоты" (Pilots List View)

## Обзор страницы

**Страница:** Список пилотов (Pilots List View)
**Путь в макете:** Боковое меню → "Пилоты"
**Статус:** Адаптация существующей страницы

### Что показывает страница (из макета):

**Табличное представление со столбцами:**
- ID (PLT-XXX)
- Код пилота
- Название пилота
- Режим пилота (A/B тест/A/B/C тест/и т.д.)
- Статус пилота (draft/requires_activation/approval_rejected/ratification_rejected/active/completed/archived)
- Статус разметки (Да/Нет)
- Размеченные заявки (количество)
- Планируемая дата активации
- Планируемая дата выключения

**Функционал:**
- Фильтрация по продукту (dropdown)
- Фильтрация по статусу
- Поиск по названию
- Сортировка по столбцам
- Пагинация
- Клик на строку → переход к детальному просмотру

**Права доступа:**
- ПРМ видит пилоты по всем продуктам (read-only для чужих), но создавать/редактировать/отправлять на согласование может только пилоты своего продукта
- Методолог видит все пилоты
- Админ видит все пилоты

## BE-P1: Backend для страницы "Пилоты"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 2 дня

### Summary
Адаптация API endpoint для списка пилотов с фильтрацией, пагинацией и правами доступа

### Description

Адаптировать Backend API для страницы списка пилотов:

1. **API Endpoint: GET /api/pilots/**

   **Параметры запроса:**
   - `product_id` (optional) - фильтр по продукту
   - `status` (optional) - фильтр по статусу (draft/requires_activation/approval_rejected/ratification_rejected/active/completed/archived)
   - `search` (optional) - поиск по названию (case-insensitive, partial match)
   - `page` (optional, default=1) - номер страницы
   - `page_size` (optional, default=20) - размер страницы
   - `ordering` (optional, default='-created_at') - сортировка

   **Формат ответа:**
   ```json
   {
     "count": 42,
     "next": "http://api/pilots/?page=2",
     "previous": null,
     "results": [
       {
         "id": "uuid",
         "display_id": "PLT-001",
         "code": "PLT-PREMIUM-001",
         "name": "Пилот оптимизации лимитов Premium карт",
         "mode": "ab_test",
         "status": "active",
         "markup_status": true,
         "marked_applications_count": 1250,
         "planned_start_at": "2026-03-10",
         "planned_end_at": "2026-04-10",
         "created_at": "2026-03-01T10:00:00Z",
         "created_by": {
           "id": "user-uuid",
           "full_name": "Иван Иванов"
         }
       }
     ]
   }
   ```

2. **Бизнес-логика:**
   - Фильтрация по правам доступа:
     - ПРМ видит все (read-only для чужих)
     - Методолог видит все
     - Админ видит все
   - Поиск по названию (case-insensitive, ILIKE для PostgreSQL)
   - Сортировка по умолчанию: -created_at (новые первые)

3. **Права доступа:**
   - Middleware проверяет роль пользователя
   - Для list/read endpoints ПРМ может читать все продукты; ограничения применяются к операциям изменения (create/update/delete/submit)

4. **Оптимизация:**
   - Select related для created_by
   - Индексы на product_id, status, created_at

### Acceptance Criteria

- [ ] GET /api/pilots/ endpoint адаптирован
- [ ] Фильтрация по product_id, status работает
- [ ] Поиск по названию работает (case-insensitive)
- [ ] Пагинация работает корректно (default page_size=20)
- [ ] Сортировка по умолчанию: -created_at
- [ ] Права доступа реализованы (ПРМ читает все продукты; изменения только для своего продукта)
- [ ] Select related оптимизация применена
- [ ] Unit тесты покрывают все сценарии (coverage > 80%)
- [ ] API документация обновлена

### Dependencies
- AN-P1 (требуется API контракт)

### Technical Notes

**Пример SQL оптимизации (Django ORM):**
```python
from django.db.models import Q

Pilot.objects.filter(
    Q(name__icontains=search),
    product_id=product_id,
    status=status
).select_related(
    'created_by'
).order_by('-created_at')
```

## FE-P1: Frontend для страницы "Пилоты"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 2 дня

### Summary
Адаптация UI страницы списка пилотов с таблицей, фильтрами и поиском

### Description

Адаптировать Frontend для страницы списка пилотов:

1. **Компонент PilotsView:**
   - Табличное представление (Material-UI Table)
   - Столбцы: ID, Код пилота, Название, Режим, Статус пилота, Статус разметки, Размеченные заявки, Планируемая дата активации, Планируемая дата выключения
   - Responsive дизайн

2. **Фильтры и поиск:**
   - Dropdown фильтр по продукту
   - Dropdown фильтр по статусу
   - Поле поиска по названию (debounce 300ms)
   - Кнопка "Сбросить фильтры"

3. **Сортировка:**
   - По умолчанию: по дате создания (новые первые)
   - Клик на заголовок столбца → сортировка
   - Индикатор направления сортировки

4. **Пагинация:**
   - Material-UI Pagination компонент
   - Показывать "Показано X-Y из Z"
   - Выбор размера страницы (10/20/50)

   5. **Статусные бейджи:**
      - **Статус пилота:**
         - draft: серый
         - requires_activation: оранжевый
         - active: синий (анимация)
         - approval_rejected: красный
         - ratification_rejected: красный
         - completed: зеленый
         - archived: серый с opacity
   - **Статус разметки:**
     - Да: зеленый
     - Нет: серый

6. **Навигация:**
   - Клик на строку → переход к детальному просмотру
   - Breadcrumbs: "Пилоты"

7. **Loading и Error состояния:**
   - Skeleton loader при загрузке
   - Error message при ошибке API
   - Empty state если нет данных

8. **API интеграция:**
   - PilotService.getList(filters, page, pageSize, ordering)
   - Обработка ошибок (403, 404, 500)
   - Toast уведомления при ошибках
