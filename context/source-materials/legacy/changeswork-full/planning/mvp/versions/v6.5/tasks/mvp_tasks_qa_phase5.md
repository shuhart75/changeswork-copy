# QA задачи для Фазы 5: Сложные страницы

## QA-PHASE5: Тестирование Фазы 5

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 3 дня

### Summary
Тестирование сложных страниц Пакеты и Согласования с batch операциями

### Description

Провести полное тестирование UI + интеграций для "сложных" страниц MVP v6.3:

**Страницы для тестирования:**
1. Страница Пакеты (FE-PKG1) - 4 вкладки, batch операции
2. Страница Согласования (FE-APR1) - 2 вкладки, решения approve/reject/ratify
3. Страница Цепочки (FE-D1) - список цепочек (lineage) + переходы в деталку внедрения

**Тестовые сценарии:**

**1. Страница Цепочки (0.5 дня):**
- Загрузка списка (GET /api/chains/)
- Колонки: Deployment, кол-во Simulations/Pilots, критичность/приоритет, статус, дата
- Фильтры: product/status/priority + search
- Переход в детальную страницу Deployment и проверка отображения lineage (widget)

**2. Страница Пакеты (1.5 дня):**

**Вкладка "Ожидают утверждения":**
- Отображение Pilot/Deployment с `awaiting_ratification=true`
- (опционально) Группировка по продукту
- Checkbox для выбора элементов
- "Отправить на утверждение (N)" + выбор ratifier + предпросмотр брифа
- Создание Package и запуск ratification

**Вкладка "На согласовании/утверждении":**
- Элементы с активным `ApprovalInstance` (Approval или Ratification)
- Отзыв (recall) выбранных элементов и корректное изменение статусов

**Вкладка "Отклонено":**
- Элементы, где последний `ApprovalInstance.status=rejected`
- Отображение причины/комментария

**Вкладка "История":**
- Список Package + завершенных ApprovalInstance (аудит)

**3. Страница Согласования (1.0 дня):**

**Вкладка "На согласовании" (Approval):**
- Назначения текущего пользователя по `ApprovalStage(stage_type=approval)`
- Типы элементов: Simulation/Pilot/Deployment/Package
- Approve/Reject (комментарий обязателен при reject)

**Вкладка "На утверждении" (Ratification):**
- Назначения текущего пользователя по `ApprovalStage(stage_type=ratification)`
- Ratify/Reject

**Интеграционное тестирование:**
- Создание Simulation/Pilot/Deployment и отправка на согласование
- Прохождение approval этапов разными пользователями
- Ветка `auto_ratification=true` (индивидуальная ratification) и `auto_ratification=false` (попадание в awaiting_ratification + Package)
- Проверка, что статусы target-сущностей синхронизируются через `BE-INT1`
- Проверка уведомлений на каждом этапе
