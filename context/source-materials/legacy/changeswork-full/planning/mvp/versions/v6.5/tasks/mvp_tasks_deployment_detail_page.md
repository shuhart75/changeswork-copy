# Задачи для детальной страницы Внедрения (Deployment Detail)

## Обзор страницы

**Страница:** Детальная внедрения (Deployment Detail)
**Путь в макете:** Клик на внедрение из списка → детальная страница
**Статус:** Новая разработка

### Что показывает страница (из макета):

**Основная информация:**
- ID внедрения (display_id)
- Название
- Статус (draft/requires_approval/approval_rejected/ratification_rejected/approved/deployed/rolled_back)
- Продукт
- Связанные скоркарты (многие-ко-многим)
- Критичность (автоматически рассчитывается как MAX критичности всех скоркарт)
- Даты (создания, одобрения, внедрения, отката)
- Автор

**Блоки:**
- **Блок артефактов** - список прикрепленных файлов (документы, результаты)
- **Блок связанных сущностей** - связи с Симуляциями и Пилотами (через Скоркарты)
- **Блок истории внедрений** - история deployed/rolled_back

**Функционал:**
- Просмотр всех полей внедрения
- Загрузка/удаление артефактов
- Просмотр связанных Симуляций (источники скоркарт)
- Просмотр связанных Пилотов (через общие скоркарты)
- Просмотр истории внедрений
- Переход к связанным сущностям (клик → детальная)
- Кнопки действий по жизненному циклу (если есть права)
- Управление версиями:
  - Просмотр текущей версии и истории версий
  - "Создать новую версию" (из текущей версии)

**Права доступа:**
- ПРМ видит внедрения по всем продуктам (read-only для чужих), но выполнять действия ЖЦ (submit/deploy/rollback) и создавать/редактировать версии может только внедрения своего продукта
- Методолог видит все внедрения
- Админ видит все внедрения

## AN-CHD1: Аналитика детальной страницы Внедрения

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 5 дней

### Summary
Детализация требований для разработки детальной страницы Внедрения с артефактами, связанными сущностями и историей внедрений

### Description

Провести аналитику для разработки детальной страницы Внедрения:

1. **Детализация бизнес-требований:**
   - Определить правила отображения артефактов
   - Уточнить логику связей через Скоркарты
   - Определить правила доступа к артефактам
   - Уточнить отображение источников скоркарт (Симуляции, Пилоты)
   - Определить формат истории внедрений
   - Определить кнопки действий по жизненному циклу

2. **Спецификация UI/UX:**
   - Дизайн блока артефактов (список, загрузка, удаление)
   - Дизайн блока связанных сущностей (Симуляции, Пилоты)
   - Дизайн блока истории внедрений
   - Спецификация навигации к связанным сущностям
   - Дизайн кнопок действий (Отправить на согласование, Внедрить, Откатить)

3. **Спецификация API контракта:**
   - GET /api/deployments/{id}/ - детальная информация
   - GET /api/deployments/{id}/artifacts/ - список артефактов
   - POST /api/deployments/{id}/artifacts/ - загрузка артефакта
   - DELETE /api/deployments/{id}/artifacts/{artifact_id}/ - удаление артефакта
   - GET /api/deployments/{id}/related/ - связанные сущности
   - GET /api/deployments/{id}/deployment_history/ - история внедрений

### Acceptance Criteria

- [ ] Бизнес-требования задокументированы
- [ ] API контракт задокументирован
- [ ] UI/UX спецификация готова
- [ ] Правила доступа определены
- [ ] Формат истории внедрений определен
- [ ] Документ согласован с заказчиком

Нет

## FE-CHD1: Frontend для детальной страницы Внедрения

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 6 дней

### Summary
Разработка UI детальной страницы Внедрения с артефактами, связанными сущностями и историей внедрений

### Description

Разработать Frontend для детальной страницы Внедрения:

1. **Компонент DeploymentDetailView:**
   - Отображение основной информации изменения
   - Блок связанных скоркарт
   - Блок артефактов
   - Блок связанных сущностей
   - Блок истории внедрений
   - Кнопки действий по жизненному циклу
   - Responsive дизайн

2. **Основная информация:**
   - ID изменения (display_id)
   - Название
   - Статус (бейдж с цветом)
   - Внедрение (кликабельная ссылка)
   - Продукт
   - Критичность (бейдж, автоматически рассчитывается)
   - Даты (создания, одобрения, внедрения, отката)
   - Автор

3. **Блок связанных скоркарт:**
   - Заголовок "Скоркарты (N)"
   - Список скоркарт:
     - ID, название, критичность
     - Клик → переход к детальной скоркарты
   - EntityScorecardBlock компонент
   - Отображение критичности изменения = MAX(критичности скоркарт)

4. **Блок артефактов:**
   - Заголовок "Артефакты (N)"
   - Список файлов:
     - Иконка типа файла
     - Название файла (кликабельное для скачивания)
     - Размер файла
     - Дата загрузки
     - Автор загрузки
     - Кнопка "Удалить" (если есть права)
   - Кнопка "Загрузить файл" (если есть права)
   - Drag & drop зона для загрузки
   - Валидация: размер <= 50MB, форматы (pdf, xlsx, docx, txt, csv)
   - Progress bar при загрузке
   - Empty state если нет артефактов

5. **Блок связанных сущностей:**
   - Заголовок "Связанные сущности"
   - Секция "Симуляции (N)" - источники скоркарт:
     - Список симуляций
     - ID, название, статус
     - "через скоркарты: SC-001, SC-002"
     - Клик → переход к детальной симуляции
   - Секция "Пилоты (N)" - через общие скоркарты:
     - Список пилотов
     - ID, название, статус
     - "через скоркарты: SC-001, SC-002"
     - Клик → переход к детальной пилота
   - Empty state если нет связей

6. **Блок истории внедрений:**
   - Заголовок "История внедрений"
   - Timeline с событиями:
     - Дата и время
     - Действие (deployed/rolled_back)
     - Пользователь
     - Причина (для rolled_back)
   - Визуализация: вертикальная timeline с иконками
   - Empty state если нет истории

7. **Кнопки действий по жизненному циклу:**
   - "Отправить на согласование" (если status = requires_approval)
   - "Внедрить" (если status = approved)
   - "Откатить" (если status = deployed)
   - Видимость кнопок зависит от прав доступа

8. **Диалог загрузки файла:**
   - Drag & drop зона
   - Кнопка "Выбрать файл"
   - Отображение выбранного файла
   - Валидация формата и размера
   - Progress bar
   - Кнопки: "Отмена", "Загрузить"

9. **Диалог отката (Rollback):**
   - Поле ввода причины отката (обязательное)
   - Предупреждение о последствиях
   - Кнопки: "Отмена", "Откатить"

10. **Loading и Error состояния:**
    - Skeleton loader при загрузке
    - Error message при ошибке API
    - Toast уведомления при успешной загрузке/удалении

11. **API интеграция:**
    - DeploymentService.getDetail(id)
    - DeploymentService.getArtifacts(id)
    - DeploymentService.uploadArtifact(id, file)
    - DeploymentService.deleteArtifact(id, artifactId)
    - DeploymentService.getRelated(id)
    - DeploymentService.getDeploymentHistory(id)
    - Обработка ошибок (403, 404, 500)
