# QA задачи для Фазы 2: Новые списки и формы

## QA-PHASE2: Тестирование Фазы 2

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 10 дней

### Summary
Комплексное тестирование новых списков (Changes, Scorecards) и всех форм создания

### Description

Провести полное тестирование функционала Фазы 2:

**Новые списки для тестирования:**
1. Список изменений (FE-C1)
2. Список скоркарт (FE-SC1) - с Excel экспортом

**Формы для тестирования:**
3. Форма симуляции (FE-SF1) - простая
4. Форма пилота (FE-PF1) - с множественным выбором скоркарт
5. Форма создания внедрения (FE-CD1) - уже протестирована в QA-CD1

**Тестовые сценарии:**

**1. Список изменений (0.5 дня):**
- Отображение данных (ID, название, статус, внедрение, продукт, критичность, даты)
- Пагинация, сортировка, фильтрация
- Фильтр по статусу (draft/requires_approval/approved/deployed/rolled_back)
- Фильтр по критичности (low/medium/high/critical)
- Фильтр по продукту
- Поиск по названию и ID
- Переход к детальной странице
- Цветовая индикация статусов
- Права доступа (ПРМ видит свой продукт, Методолог/Админ видят все)

**2. Список скоркарт (0.5 дня):**
- Отображение данных (ID, название, версия, продукт, подпродукты, критичность, шаблон)
- Пагинация, сортировка, фильтрация
- Фильтр по продукту
- Фильтр по критичности
- Фильтр по шаблону
- Поиск по названию и ID
- Excel экспорт:
  - Кнопка "Экспорт в Excel"
  - Экспорт текущей страницы
  - Экспорт всех записей (с учетом фильтров)
  - Формат файла (.xlsx)
  - Корректность данных в файле
  - Имя файла с датой/временем
- Переход к детальной странице
- Права доступа

**3. Форма симуляции (0.25 дня):**
- Поля: название (обязательное), внедрение (select, обязательное), описание (опционально)
- Валидация обязательных полей
- Dropdown внедрений (только активные)
- Кнопки: "Отмена", "Создать"
- Создание симуляции
- Редирект на детальную страницу после создания
- Error handling
- Права доступа (ПРМ, Методолог, Админ)

**4. Форма пилота (0.5 дня):**
- Поля: название (обязательное), внедрение (select, обязательное), скоркарты (multi-select, обязательное минимум 1), описание
- Валидация: минимум 1 скоркарта должна быть выбрана
- Multi-select скоркарт с поиском
- Отображение выбранных скоркарт (chips с возможностью удаления)
- Фильтрация скоркарт по продукту внедрения
- Кнопки: "Отмена", "Создать"
- Создание пилота
- Редирект на детальную страницу после создания
- Error handling
- Права доступа

**5. Интеграционное тестирование (0.25 дня):**
- Создание симуляции → переход к детальной → создание скоркарты из результатов
- Создание пилота → выбор скоркарт → проверка связей на детальной
- Создание внедрения → создание симуляции → создание пилота → проверка связей

### Acceptance Criteria

- [ ] Список изменений отображается корректно
- [ ] Фильтрация по статусу и критичности работает
- [ ] Список скоркарт отображается корректно
- [ ] Excel экспорт работает (текущая страница и все записи)
- [ ] Экспортированный файл содержит корректные данные
- [ ] Форма симуляции: валидация работает
- [ ] Форма симуляции: создание работает
- [ ] Форма пилота: multi-select скоркарт работает
- [ ] Форма пилота: валидация минимум 1 скоркарты работает
- [ ] Форма пилота: фильтрация скоркарт по продукту работает
- [ ] Форма пилота: создание работает
- [ ] Редирект после создания работает
- [ ] Связи между сущностями устанавливаются корректно
- [ ] Права доступа работают корректно
- [ ] Error handling работает
- [ ] Все найденные баги задокументированы
- [ ] Критические баги исправлены

### Dependencies
- FE-C1 (список изменений)
- FE-SC1 (список скоркарт)
- FE-SF1 (форма симуляции)
- FE-PF1 (форма пилота)
- QA-CD1 (тестирование формы создания внедрения)

### Deliverables
- Отчет о тестировании
- Список найденных багов
- Примеры экспортированных Excel файлов
