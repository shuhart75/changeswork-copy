# QA задачи для Фазы 5: Сложные страницы

## QA-PHASE5: Тестирование Фазы 5

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 3 дня

### Summary
Тестирование сложных страниц Пакеты и Согласования с batch операциями

### Description

Провести полное тестирование функционала Фазы 5:

**Страницы для тестирования:**
1. Страница Пакеты (FE-PKG1) - 4 вкладки, batch операции
2. Страница Согласования (FE-APR1) - 2 вкладки, batch операции

**Тестовые сценарии:**

**1. Страница Пакеты (1.5 дня):**

**Вкладка "Черновики":**
- Отображение цепочек в статусе draft
- Группировка по Initiative
- Колонки: ID цепочки, Initiative, Симуляции, Пилоты, Внедрение, Критичность, Дата создания
- Сортировка по всем колонкам
- Фильтрация по Initiative, критичности
- Поиск по ID и названию
- Checkbox для выбора цепочек
- Кнопка "Отправить на согласование" (batch):
  - Активна только если выбрано >= 1 цепочки
  - Валидация: все выбранные цепочки должны иметь одинаковый маршрут согласования
  - Диалог выбора маршрута согласования
  - Назначение согласующих для этапа Approval
  - Назначение утверждающих для этапа Ratification
  - Создание Package
  - Предпросмотр email брифа
  - Кнопки: "Отмена", "Отправить"
- Переход к детальной цепочки (клик на строку)

**Вкладка "На согласовании":**
- Отображение цепочек в процессе согласования (этап Approval)
- Группировка по Package
- Колонки: ID Package, Цепочки (количество), Критичность (MAX), Согласующие, Дата отправки, Статус
- Фильтрация по назначениям (мои/все)
- Отображение прогресса согласования (N из M согласовали)
- Раскрытие Package → список цепочек
- Переход к детальной цепочки

**Вкладка "На утверждении":**
- Отображение цепочек в процессе утверждения (этап Ratification)
- Аналогично вкладке "На согласовании"
- Отображение прогресса утверждения

**Вкладка "Завершенные":**
- Отображение цепочек, прошедших согласование (approved) или отклоненных
- Группировка по Package
- Колонки: ID Package, Цепочки, Критичность, Результат (approved/rejected), Дата завершения
- Фильтрация по результату
- История согласования (кто, когда, решение, комментарий)

**Batch операции:**
- Выбор нескольких цепочек (checkbox)
- Отправка на согласование (только с одинаковым маршрутом)
- Валидация перед отправкой
- Создание Package
- Предпросмотр email брифа для всех согласующих

**2. Страница Согласования (1.5 дня):**

**Вкладка "На согласовании" (Approval):**
- Отображение Packages, где текущий пользователь назначен согласующим
- Группировка по Package
- Колонки: ID Package, Цепочки (количество), Критичность (MAX), Автор, Дата отправки, Мой статус
- Фильтрация по статусу (pending/approved/rejected)
- Раскрытие Package → список цепочек с деталями
- Для каждой цепочки:
  - Основная информация (Deployment, Симуляции, Пилоты, Изменение)
  - Кнопки: "Согласовать", "Отклонить"
  - Поле комментария (обязательно при отклонении)
- Batch согласование:
  - Checkbox для выбора нескольких цепочек
  - Кнопка "Согласовать выбранные"
  - Кнопка "Отклонить выбранные" (с общим комментарием)
- Отображение решений других согласующих
- Прогресс согласования (N из M)

**Вкладка "На утверждении" (Ratification):**
- Отображение Packages, где текущий пользователь назначен утверждающим
- Аналогично вкладке "На согласовании"
- Доступна только после прохождения этапа Approval
- Batch утверждение/отклонение

**Интеграционное тестирование:**
- Создание нескольких цепочек с одинаковым маршрутом
- Отправка на согласование (batch) → создание Package
- Согласование от имени согласующего
- Утверждение от имени утверждающего
- Проверка перехода статусов всех цепочек
- Проверка уведомлений на каждом этапе
- Отклонение → проверка возврата в draft
- Частичное согласование (один согласующий одобрил, другой отклонил)
