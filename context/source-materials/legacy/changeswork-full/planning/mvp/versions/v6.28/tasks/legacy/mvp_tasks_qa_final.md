# QA задача финального тестирования MVP

## QA-FINAL: Финальное интеграционное тестирование MVP

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 3 дня

### Summary
Полное end-to-end тестирование всего MVP перед релизом

### Description

Провести финальное комплексное тестирование всей системы:

**Тестовые сценарии:**

**1. End-to-End сценарии (1.5 дня):**

**Сценарий 1: Полный цикл внедрения**
1. Создание Deployment
2. Создание Simulation в Deployment
3. Запуск Simulation (draft → running → completed)
4. Создание Scorecard из результатов Simulation
5. Создание Pilot с выбором Scorecard
6. Отправка Pilot на активацию
7. Согласование Pilot (Approval → Ratification)
8. Активация Pilot (requires_activation → active)
9. Завершение Pilot (active → completed)
10. Создание Deployment с выбором Scorecard из Pilot
11. Отправка Deployment на согласование
12. Согласование Deployment (Approval → Ratification)
13. Утверждение Deployment (requires_approval → approved)
14. Внедрение Deployment (approved → deployed)
15. Проверка всех связей на детальных страницах
16. Проверка всех уведомлений на каждом этапе

**Сценарий 2: Batch операции**
1. Создание нескольких Deployments
2. Создание нескольких Simulations
3. Создание нескольких Scorecards
4. Создание нескольких Pilots с одинаковым маршрутом
5. Batch отправка Pilots на согласование → создание Package
6. Batch согласование Package
7. Batch утверждение Package
8. Проверка активации всех Pilots
9. Создание нескольких Changes
10. Batch отправка Changes на согласование
11. Batch согласование и утверждение
12. Проверка всех уведомлений

**Сценарий 3: Rollback**
1. Создание и внедрение Deployment (deployed)
2. Rollback Deployment с указанием причины
3. Проверка истории внедрений на детальной странице
4. Проверка уведомлений о rollback
5. Создание нового Deployment для исправления

**Сценарий 4: Отклонение на согласовании**
1. Создание Pilot
2. Отправка на согласование
3. Отклонение на этапе Approval с комментарием
4. Проверка возврата в draft
5. Исправление и повторная отправка
6. Согласование на Approval
7. Отклонение на этапе Ratification
8. Проверка возврата в draft
9. Повторная отправка и успешное прохождение

**2. Регрессионное тестирование (0.5 дня):**
- Повторное прохождение всех тест-кейсов из QA-PHASE1 до QA-PHASE5
- Проверка, что ничего не сломалось после интеграции
- Проверка производительности на больших объемах данных

**3. Тестирование производительности (0.5 дня):**
- Загрузка 1000+ записей в списки
- Проверка пагинации на больших объемах
- Проверка сортировки и фильтрации
- Проверка поиска
- Загрузка больших файлов (до 50MB)
- Batch операции с 50+ записями
- Время отклика API (должно быть < 2 сек)
- Время загрузки страниц (должно быть < 3 сек)

**4. Тестирование безопасности (0.5 дня):**
- Проверка прав доступа:
  - ПРМ видит все продукты (read-only для чужих), но не может изменять/отправлять на согласование чужие элементы
  - Методолог видит все
  - Согласующий видит только свои назначения
  - Админ видит все
- Попытки доступа к чужим данным (должны блокироваться)
- Валидация загружаемых файлов (размер, формат)
- XSS защита в полях ввода
- CSRF защита
- SQL injection защита

**5. Кросс-браузерное тестирование:**
- Chrome (последняя версия)
- Firefox (последняя версия)
- Safari (последняя версия)
- Edge (последняя версия)
- Проверка на разных разрешениях (desktop, tablet, mobile)

**6. Тестирование доступности (accessibility):**
- Навигация с клавиатуры (Tab, Enter, Esc)
- Screen reader совместимость
- Контрастность цветов (WCAG AA)
- Альтернативные тексты для изображений
- ARIA атрибуты

### Acceptance Criteria

- [ ] Все end-to-end сценарии проходят успешно
- [ ] Полный цикл внедрения работает от начала до конца
- [ ] Batch операции работают корректно
- [ ] Rollback работает корректно
- [ ] Отклонение на согласовании работает корректно
- [ ] Регрессионные тесты пройдены
- [ ] Производительность соответствует требованиям
- [ ] Время отклика API < 2 сек
- [ ] Время загрузки страниц < 3 сек
- [ ] Права доступа работают корректно
- [ ] Безопасность: нет уязвимостей
- [ ] Кросс-браузерная совместимость подтверждена
- [ ] Accessibility требования выполнены (WCAG AA)
- [ ] Все уведомления отправляются корректно
- [ ] Все связи между сущностями работают
- [ ] Все найденные баги задокументированы
- [ ] Все критические и высокие баги исправлены
- [ ] Средние и низкие баги задокументированы для будущих релизов
