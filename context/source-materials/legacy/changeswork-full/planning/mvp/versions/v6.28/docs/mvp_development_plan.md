## Фаза 3: Backend процессы (Дни 18-45, критический путь)

### Цель
Реализовать жизненные циклы и процесс согласования

### Последовательный поток (24 дня)

```
BE-CL1 (5д) → BE-AP1 (8д) → BE-INT1 (3д) → BE-PL1 (3д)
                                          ↘
                                           BE-NOT1 (5д, параллельно)
```

**Независимо:**
```
BE-SL1 (2д) - можно в любое время
```

### Детали

**BE-CL1 (5 дней) - ЖЦ изменения:**
- Состояния: draft → requires_approval → approved → deployed → rolled_back
- Валидация минимум 1 скоркарты
- Автоматический расчет критичности

**BE-AP1 (8 дней) - Процесс согласования:**
- Двухэтапный процесс (Approval → Ratification)
- Универсальный для Pilot, Change, Package
- Маршруты согласования

**BE-INT1 (3 дня) - Интеграция ЖЦ:**
- Pilot: requires_activation → active
- Change: requires_approval → approved
- Автоматические переходы при согласовании/отклонении

**BE-PL1 (3 дня) - ЖЦ пилота:**
- Состояния: draft → requires_activation → active → completed
- Интеграция с процессом согласования

**BE-NOT1 (5 дней) - Уведомления:**
- Email уведомления для всех событий
- Шаблоны с брендингом
- История уведомлений

**BE-SL1 (2 дня) - ЖЦ симуляции:**
- Состояния: draft → running → completed/failed
- Независимый от согласования

### Результаты фазы
- ✅ Все жизненные циклы реализованы
- ✅ Процесс согласования работает
- ✅ Уведомления отправляются
- ✅ Интеграция завершена

### QA
- **QA-PHASE3 (3 дня):** Интеграционное тестирование всех backend процессов, жизненных циклов и уведомлений

**4. Форма скоркарты (10 дней)**

- Риск: Сложная логика связей Simulations → Scorecards → Pilots → Changes

**6. Детальная изменения (13 дней)**
