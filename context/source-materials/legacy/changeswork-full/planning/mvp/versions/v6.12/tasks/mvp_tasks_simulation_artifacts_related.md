## Обзор

**Компонент:** Симуляции (страницы + формы)  
**Статус:** Доработка существующего функционала (MVP v6.8)  

### MVP-правило

- Симуляции **не участвуют** в approval/ratification, их жизненный цикл не меняем.
- В MVP добавляем/унифицируем UI-блоки:
  - **Артефакты**: ссылки (URL) на внешние документы (через `BE-ART1`).
  - **Связанные сущности**: read-only связи с Пилотами и Внедрениями (через скоркарты/существующие связи).

## FE-SA1: Frontend блоков "Артефакты" и "Связанные сущности" на Симуляции

**Тип:** Task  
**Приоритет:** Высокий  
**Оценка:** 3 дня (подтверждено)  

### Scope

1. **Simulation detail**
   - Добавить блок **Артефакты**:
     - список артефактов (name/type/url)
     - добавление/удаление (при наличии прав)
     - открыть по ссылке
   - Добавить блок **Связанные сущности** (read-only):
     - Пилоты (через скоркарты)
     - Внедрения (через скоркарты)

2. **Simulation create/edit form**
   - Блоки присутствуют в UI.
   - В режиме **create**:
     - Артефакты: disabled/empty c текстом "Доступно после создания симуляции".
     - Связанные сущности: disabled/empty c текстом "Доступно после создания симуляции".
   - В режиме **edit**:
     - Артефакты: включены (list/add/delete).
     - Связанные сущности: включены (read-only).

### API интеграция

- Артефакты (через `BE-ART1`):
  - `GET /api/simulations/{id}/artifacts/` или `GET /api/artifacts/?entity_type=simulation&entity_id=...`
  - `POST /api/simulations/{id}/artifacts/` или `POST /api/artifacts/`
  - `DELETE /api/simulations/{id}/artifacts/{artifact_id}/` или `DELETE /api/artifacts/{id}/`
- Связанные сущности:
  - использовать существующий `GET /api/simulations/{id}/related/` (если он уже есть в проекте)
  - если эндпоинта нет, зафиксировать как backend-gap (не в оценке `FE-SA1`)

### Acceptance Criteria

- [ ] На detail симуляции отображаются блоки "Артефакты" и "Связанные сущности"
- [ ] В edit-форме симуляции артефакты добавляются/удаляются (URL) с учетом прав
- [ ] В create-форме блоки присутствуют и корректно disabled/empty
- [ ] Связанные сущности отображаются read-only (или корректный empty state)

### Dependencies

- `BE_ART1` (артефакты как URL)
