# QA задачи для Фазы 1: Базовые списки и детальные страницы

## QA-PHASE1: Тестирование Фазы 1

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 10 дней

### Summary
Комплексное тестирование всех списков и детальных страниц Фазы 1

### Description

Провести полное тестирование функционала Фазы 1:

**Списки для тестирования:**
1. Список внедрений (FE-D1)
2. Список симуляций (FE-S1)
3. Список пилотов (FE-P1)

**Детальные страницы для тестирования:**
4. Детальная внедрения (FE-DD1) - 4 вкладки, 3 inline формы
5. Детальная симуляции (FE-SD1) - артефакты, связанные сущности
6. Детальная пилота (FE-PD1) - артефакты, связанные сущности, скоркарты

**Тестовые сценарии:**

**1. Списки (1 день):**
- Отображение данных в таблицах
- Пагинация (10/25/50/100 записей)
- Сортировка по всем колонкам
- Фильтрация по всем полям
- Поиск по названию и ID
- Переход к детальной странице (клик на строку)
- Responsive дизайн (desktop, tablet, mobile)
- Loading состояния
- Empty states
- Error handling

**2. Детальная внедрения (1 день):**
- Отображение основной информации
- Вкладка "Симуляции": список, переходы
- Вкладка "Пилоты": список, переходы
- Вкладка "Изменения": список, переходы
- Вкладка "Артефакты": добавление ссылки, открытие ссылки, удаление
- Inline форма создания симуляции
- Inline форма создания пилота
- Inline форма создания изменения
- Валидация форм
- Кнопка "Архивировать" (если нет deployed изменений)
- Права доступа (ПРМ, Методолог, Админ)

**3. Детальные симуляции и пилота (1 день):**
- Отображение основной информации
- Блок артефактов: добавление ссылки (name/type/url), открытие ссылки, удаление
- Валидация URL (формат) и обязательных полей (name/url)
- Блок связанных сущностей:
  - Для симуляции: связанные пилоты и изменения через скоркарты
  - Для пилота: связанные симуляции (источники) и изменения через скоркарты
- Навигация к связанным сущностям (клик → детальная)
- Отображение "через скоркарты: SC-001, SC-002"
- Empty states
- Права доступа

**4. Кросс-браузерное тестирование:**
- Chrome (последняя версия)
- Firefox (последняя версия)
- Safari (последняя версия)
- Edge (последняя версия)

**5. Регрессионное тестирование:**
- Проверка, что существующий функционал не сломался

### Acceptance Criteria

- [ ] Все списки отображаются корректно
- [ ] Пагинация работает на всех списках
- [ ] Сортировка работает по всем колонкам
- [ ] Фильтрация работает по всем полям
- [ ] Поиск находит записи по названию и ID
- [ ] Переход к детальной странице работает
- [ ] Детальная внедрения: все 4 вкладки работают
- [ ] Детальная внедрения: все 3 inline формы работают
- [ ] Детальная симуляции: артефакты и связанные сущности работают
- [ ] Детальная пилота: артефакты, скоркарты и связанные сущности работают
- [ ] Добавление ссылок на артефакты работает
- [ ] Валидация URL работает
- [ ] Открытие ссылок работает
- [ ] Удаление файлов работает (с проверкой прав)
- [ ] Навигация к связанным сущностям работает
- [ ] Отображение "через скоркарты" корректно
- [ ] Empty states отображаются
- [ ] Loading состояния работают
- [ ] Error handling работает
- [ ] Права доступа работают корректно
- [ ] Responsive дизайн работает на всех разрешениях
- [ ] Кросс-браузерная совместимость подтверждена
- [ ] Регрессионные тесты пройдены
- [ ] Все найденные баги задокументированы
- [ ] Критические баги исправлены

### Dependencies
- FE-DD1 (детальная внедрения должна быть готова)
- FE-SD1 (детальная симуляции должна быть готова)
- FE-PD1 (детальная пилота должна быть готова)

### Deliverables
- Отчет о тестировании с результатами всех тест-кейсов
- Список найденных багов с приоритетами
- Скриншоты/видео для критических багов
- Рекомендации по улучшению UX (если есть)
