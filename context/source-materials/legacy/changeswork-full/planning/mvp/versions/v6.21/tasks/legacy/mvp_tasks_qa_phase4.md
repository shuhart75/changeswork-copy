# QA задачи для Фазы 4: Детальные и формы Change/Scorecard

## QA-PHASE4: Тестирование Фазы 4

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 2 дня

### Summary
Тестирование детальных страниц и форм для Change и Scorecard

### Description

Провести полное тестирование функционала Фазы 4:

**Детальные страницы для тестирования:**
1. Детальная изменения (FE-CHD1)
2. Детальная скоркарты (FE-SCD1)

**Формы для тестирования:**
3. Форма изменения (FE-CHF1)
4. Форма скоркарты (FE-SCF1)

**Тестовые сценарии:**

**1. Детальная изменения (1 день):**
- Отображение основной информации:
  - ID, название, статус, внедрение, продукт
  - Критичность (автоматически = MAX критичности скоркарт)
  - Даты (создания, одобрения, внедрения, отката)
  - Автор
- Блок связанных скоркарт:
  - Список скоркарт (ID, название, критичность)
  - Клик → переход к детальной скоркарты
  - Отображение критичности изменения = MAX(критичности скоркарт)
- Блок артефактов:
  - Добавление ссылки на артефакт (name/type/url)
  - Валидация URL + обязательных полей
  - Список артефактов (название, тип, ссылка, дата, автор)
  - Открытие ссылки
  - Удаление артефакта (если есть права)
  - Empty state
- Блок связанных сущностей:
  - Секция "Симуляции" - источники скоркарт
  - Секция "Пилоты" - через общие скоркарты
  - Отображение "через скоркарты: SC-001, SC-002"
  - Клик → переход к детальной сущности
  - Empty state
- Блок истории внедрений:
  - Timeline с событиями (deployed/rolled_back)
  - Дата, время, действие, пользователь
  - Причина (для rolled_back)
  - Визуализация timeline
  - Empty state
- Кнопки действий по ЖЦ:
  - "Отправить на согласование" (если status = requires_approval)
  - "Внедрить" (если status = approved)
  - "Откатить" (если status = deployed)
  - Видимость зависит от прав доступа
- Диалог отката:
  - Поле причины отката (обязательное)
  - Предупреждение о последствиях
  - Кнопки: "Отмена", "Откатить"
- Права доступа (ПРМ свой продукт, Методолог, Админ)

**2. Детальная скоркарты (0.5 дня):**
- Отображение основной информации:
  - ID, название, версия, продукт, подпродукты
  - Критичность, шаблон, конфигурация (JSON viewer)
  - Финансовые эффекты, даты, автор
- Блок источников:
  - Секция "Симуляции" - откуда создана скоркарта
  - Секция "Пилоты" - откуда создана скоркарта
  - Клик → переход к детальной
  - Empty state
- Блок использования:
  - Секция "Пилоты" - где используется
  - Секция "Изменения" - где используется
  - Клик → переход к детальной
  - Empty state
- Блок артефактов (аналогично детальной изменения)
- Права доступа (ПРМ свой продукт, Методолог, Админ)

**3. Форма изменения (0.25 дня):**
- Поля:
  - Название (обязательное)
  - Внедрение (select, обязательное)
  - Скоркарты (multi-select, обязательное минимум 1)
  - Описание (опционально)
- Автоматический расчет критичности = MAX(критичности выбранных скоркарт)
- Отображение рассчитанной критичности в реальном времени
- Multi-select скоркарт с поиском
- Фильтрация скоркарт по продукту внедрения
- Валидация минимум 1 скоркарты
- Кнопки: "Отмена", "Создать"
- Создание изменения
- Редирект на детальную страницу
- Error handling

**4. Форма скоркарты (0.25 дня):**
- Поля:
  - Название (обязательное)
  - Версия (обязательное)
  - Продукт (select, обязательное)
  - Подпродукты (multi-select, опционально)
  - Критичность (select: low/medium/high/critical, обязательное)
  - Шаблон (select, обязательное)
  - Конфигурация (JSON editor, обязательное)
  - Финансовые эффекты (JSON, опционально)
- JSON editor:
  - Подсветка синтаксиса
  - Валидация JSON
  - Автоформатирование
  - Кнопка "Загрузить из шаблона"
- Загрузка конфигурации из выбранного шаблона
- Валидация JSON
- Кнопки: "Отмена", "Создать"
- Создание скоркарты
- Редирект на детальную страницу
- Error handling
