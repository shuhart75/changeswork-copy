# Задачи для страницы "Внедрения" (Deployments List View)

## Обзор страницы

**Страница:** Список внедрений (Deployments List View)
**Путь в макете:** Боковое меню → "Внедрения"
**Статус:** Новая разработка

### Что показывает страница (из макета):

**Табличное представление со столбцами:**
- ID (DEP-XXX)
- Название
- Продукт
- Статус (draft/requires_approval/approved/deployed/rolled_back)
- Критичность (автоматический расчет: MAX критичности всех скоркарт)

- Создано (дата создания)

- Фильтрация по статусу

- Сортировка по столбцам
- Пагинация
- Клик на строку → переход к детальному просмотру

**Права доступа:**
- ПРМ видит только внедрения своего продукта
- Методолог видит все внедрения
- Админ видит все внедрения

## BE-C1: Backend для страницы "Внедрения"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 3 дня

### Summary
Разработка API endpoint для списка внедрений с фильтрацией, пагинацией и автоматическим расчетом критичности

### Description

Разработать Backend API для страницы списка внедрений:

1. **API Endpoint: GET /api/deployments/**

   **Параметры запроса:**
   - `product_id` (optional) - фильтр по продукту
   - `status` (optional) - фильтр по статусу (draft/requires_approval/approved/deployed/rolled_back)
   - `search` (optional) - поиск по названию (case-insensitive, partial match)
   - `page` (optional, default=1) - номер страницы
   - `page_size` (optional, default=20) - размер страницы
   - `ordering` (optional, default='-created_at') - сортировка

   **Формат ответа:**
   ```json
   {
     "count": 42,
     "next": "http://api/deployments/?page=2",
     "previous": null,
     "results": [
       {
         "id": "uuid",
        "display_id": "DEP-001",
         "name": "Внедрение новой скоркарты Premium",
         "deployment": {
           "id": "uuid",
           "display_id": "DEP-001",
           "name": "Оптимизация риск-стратегии Premium карт"
         },
         "status": "approved",
         "criticality": "high",
         "criticality_calculated": true,
         "created_at": "2026-03-01T10:00:00Z",
         "created_by": {
           "id": "user-uuid",
           "full_name": "Иван Иванов"
         },
         "deployed_at": "2026-03-15T14:30:00Z"
       }
     ]
   }
   ```

2. **Бизнес-логика:**
   - Автоматический расчет критичности: MAX(criticality) всех связанных скоркарт
   - Фильтрация по правам доступа:
     - ПРМ видит только свой продукт
     - Методолог видит все
     - Админ видит все
   - Поиск по названию (case-insensitive, ILIKE для PostgreSQL)
   - Сортировка по умолчанию: -created_at (новые первые)

3. **Права доступа:**
   - Middleware проверяет роль пользователя
   - Автоматическая фильтрация по product_id для ПРМ
   - HTTP 403 если ПРМ пытается получить чужой продукт

4. **Оптимизация:**
   - Select related для deployment, created_by
   - Prefetch related для scorecards (для расчета критичности)
   - Annotate для критичности (если возможно)
   - Индексы на product_id, status, created_at

- [ ] Поиск по названию работает (case-insensitive)

### Technical Notes

**Пример расчета критичности:**
```python
def calculate_criticality(deployment):
    """Рассчитать критичность как MAX критичности всех скоркарт"""
    scorecards = deployment.scorecards.all()
    if not scorecards:
        return 'low'
    
    criticality_order = {'low': 1, 'medium': 2, 'high': 3, 'critical': 4}
    max_criticality = max(
        criticality_order.get(sc.criticality, 1) 
        for sc in scorecards
    )
    
    for level, value in criticality_order.items():
        if value == max_criticality:
            return level
    
    return 'low'
```

**Пример SQL оптимизации (Django ORM):**
```python
from django.db.models import Q

Deployment.objects.filter(
    Q(name__icontains=search),
    product_id=product_id,
    status=status
).select_related(
    'deployment', 'created_by'
).prefetch_related(
    'scorecards'
).order_by('-created_at')
```

## FE-C1: Frontend для страницы "Внедрения"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 3 дня

### Summary
Разработка UI страницы списка внедрений с таблицей, фильтрами и автоматическим расчетом критичности

### Description

Разработать Frontend для страницы списка внедрений:

1. **Компонент DeploymentsView:**
   - Табличное представление (Material-UI Table)
   - Столбцы: ID, Название, Внедрение, Статус, Критичность, Создал, Создано, Развернуто
   - Responsive дизайн

2. **Фильтры и поиск:**
   - Dropdown фильтр по продукту
   - Dropdown фильтр по статусу
   - Поле поиска по названию (debounce 300ms)
   - Кнопка "Сбросить фильтры"

3. **Сортировка:**
   - По умолчанию: по дате создания (новые первые)
   - Клик на заголовок столбца → сортировка
   - Индикатор направления сортировки

4. **Пагинация:**
   - Material-UI Pagination компонент
   - Показывать "Показано X-Y из Z"
   - Выбор размера страницы (10/20/50)

5. **Статусные бейджи:**
   - draft: серый
   - requires_approval: оранжевый
   - approved: синий
   - deployed: зеленый
   - rolled_back: красный

6. **Критичность:**
   - Автоматический расчет (MAX скоркарт)
   - Цветной чип:
     - low: зеленый
     - medium: желтый
     - high: красный
     - critical: темно-красный

7. **Навигация:**
   - Клик на строку → переход к детальному просмотру
   - Breadcrumbs: "Внедрения"

8. **Loading и Error состояния:**
   - Skeleton loader при загрузке
   - Error message при ошибке API
   - Empty state если нет данных

9. **API интеграция:**
   - DeploymentService.getList(filters, page, pageSize, ordering)
   - Обработка ошибок (403, 404, 500)
   - Toast уведомления при ошибках
