# Задачи для детальной страницы Пилота (Pilot Detail)

## Обзор страницы

**Страница:** Детальная пилота (Pilot Detail)
**Путь в макете:** Клик на пилот из списка → детальная страница
**Статус:** Адаптация существующей страницы

**Примечание (v6.1/Gantt v6):** в Gantt v6 отсутствует отдельная backend-задача `BE-PD1`; план предполагает, что backend-деталка пилота уже существует (или входит в `BE-P1`). Если это не так, добавить `BE-PD1` в план.

### Что показывает страница (из макета):

**Основная информация:**
- ID пилота (display_id)
- Название
- Статус (draft/requires_activation/approval_rejected/ratification_rejected/active/completed/archived)
- Внедрение (deployment)
- Продукт
- Связанные скоркарты (многие-ко-многим)
- Даты (создания, активации, завершения)
- Автор

**Новые блоки для MVP:**
- **Блок артефактов** - список прикрепленных файлов (документы, результаты)
- **Блок связанных сущностей** - связи с Симуляциями (источники) и Внедрениями (через Скоркарты)

**Функционал:**
- Просмотр всех полей пилота
- Загрузка/удаление артефактов
- Просмотр связанных Симуляций (источники скоркарт)
- Просмотр связанных Внедрений (через общие скоркарты)
- Переход к связанным сущностям (клик → детальная)
- Кнопки действий по жизненному циклу (если есть права)
 - Управление версиями:
   - Просмотр текущей версии и истории версий
   - "Создать новую версию" для active-пилота (старая версия остаётся активной до публикации новой)

**Права доступа:**
- ПРМ видит пилоты по всем продуктам (read-only для чужих), но выполнять действия ЖЦ и редактировать может только пилоты своего продукта
- Методолог видит все пилоты
- Админ видит все пилоты

## AN-PD1: Аналитика детальной страницы Пилота

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 1 день

### Summary
Детализация требований для адаптации детальной страницы Пилота с добавлением блоков артефактов и связанных сущностей

### Description

Провести аналитику для адаптации детальной страницы Пилота:

1. **Детализация бизнес-требований:**
   - Определить правила отображения артефактов
   - Уточнить логику связей через Скоркарты
   - Определить правила доступа к артефактам
   - Уточнить отображение источников скоркарт (Симуляции)
   - Определить кнопки действий по жизненному циклу

2. **Спецификация UI/UX:**
   - Дизайн блока артефактов (список, загрузка, удаление)
   - Дизайн блока связанных сущностей (Симуляции, Внедрения)
   - Спецификация навигации к связанным сущностям
   - Дизайн кнопок действий (Отправить на согласование, Завершить)

3. **Спецификация API контракта:**
   - GET /api/pilots/{id}/ - детальная информация
   - GET /api/pilots/{id}/artifacts/ - список артефактов
   - POST /api/pilots/{id}/artifacts/ - загрузка артефакта
   - DELETE /api/pilots/{id}/artifacts/{artifact_id}/ - удаление артефакта
   - GET /api/pilots/{id}/related/ - связанные сущности

### Acceptance Criteria

- [ ] Бизнес-требования задокументированы
- [ ] API контракт задокументирован
- [ ] UI/UX спецификация готова
- [ ] Правила доступа определены
- [ ] Документ согласован с заказчиком

### Dependencies
Нет

### Deliverables
- Документ "Спецификация детальной страницы Пилота" (markdown/confluence)
- API контракт для артефактов и связанных сущностей

   **Формат ответа:**
   ```json

     "results": [

         "id": "uuid",
   ```

3. **API Endpoint: POST /api/pilots/{id}/artifacts/**

   **Параметры запроса:**
   - `file` (multipart/form-data) - файл для загрузки

   **Бизнес-логика:**
   - Валидация: размер файла <= 50MB
   - Валидация: разрешенные форматы (pdf, xlsx, docx, txt, csv)
   - Сохранение в файловое хранилище
   - Создание записи Artifact

4. **API Endpoint: DELETE /api/pilots/{id}/artifacts/{artifact_id}/**

   **Бизнес-логика:**
   - Валидация: пользователь имеет право удалять (автор или админ)
   - Удаление файла из хранилища
   - Удаление записи Artifact

5. **API Endpoint: GET /api/pilots/{id}/related/**

   **Формат ответа:**
   ```json

       {
         "id": "uuid",
   ```

6. **Права доступа:**
   - Просмотреть может: ПРМ (свой продукт), Методолог, Админ
   - Загрузить артефакт может: ПРМ (свой продукт), Методолог, Админ
   - Удалить артефакт может: автор артефакта, Админ

## FE-PD1: Frontend для детальной страницы Пилота

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 5 дней

### Summary
Адаптация UI детальной страницы Пилота с добавлением блоков артефактов и связанных сущностей

### Description

Адаптировать Frontend для детальной страницы Пилота:

1. **Компонент PilotDetailView:**
   - Отображение основной информации пилота
   - Блок связанных скоркарт
   - Блок артефактов
   - Блок связанных сущностей
   - Кнопки действий по жизненному циклу
   - Responsive дизайн

2. **Основная информация:**
   - ID пилота (display_id)
   - Название
   - Статус (бейдж с цветом)
   - Внедрение (кликабельная ссылка)
   - Продукт
   - Даты (создания, активации, завершения)
   - Автор

3. **Блок связанных скоркарт:**
   - Заголовок "Скоркарты (N)"
   - Список скоркарт:
     - ID, название, критичность
     - Клик → переход к детальной скоркарты
   - EntityScorecardBlock компонент

4. **Блок артефактов:**
   - Заголовок "Артефакты (N)"
   - Список файлов:
     - Иконка типа файла
     - Название файла (кликабельное для скачивания)
     - Размер файла
     - Дата загрузки
     - Автор загрузки
     - Кнопка "Удалить" (если есть права)
   - Кнопка "Загрузить файл" (если есть права)
   - Drag & drop зона для загрузки
   - Валидация: размер <= 50MB, форматы (pdf, xlsx, docx, txt, csv)
   - Progress bar при загрузке
   - Empty state если нет артефактов

5. **Блок связанных сущностей:**
   - Заголовок "Связанные сущности"
   - Секция "Симуляции (N)" - источники скоркарт:
     - Список симуляций
     - ID, название, статус
     - "через скоркарты: SC-001, SC-002"
     - Клик → переход к детальной симуляции
   - Секция "Внедрения (N)" - через общие скоркарты:
     - Список внедрений
     - ID, название, статус
     - "через скоркарты: SC-001, SC-002"
     - Клик → переход к детальной изменения
   - Empty state если нет связей

6. **Кнопки действий по жизненному циклу:**
  - "Отправить на согласование" (если status = requires_activation)
   - "Завершить пилот" (если status = active)
   - Видимость кнопок зависит от прав доступа

7. **Диалог загрузки файла:**
   - Drag & drop зона
   - Кнопка "Выбрать файл"
   - Отображение выбранного файла
   - Валидация формата и размера
   - Progress bar
   - Кнопки: "Отмена", "Загрузить"

8. **Loading и Error состояния:**
   - Skeleton loader при загрузке
   - Error message при ошибке API
   - Toast уведомления при успешной загрузке/удалении

9. **API интеграция:**
   - PilotService.getDetail(id)
   - PilotService.getArtifacts(id)
   - PilotService.uploadArtifact(id, file)
   - PilotService.deleteArtifact(id, artifactId)
   - PilotService.getRelated(id)
   - Обработка ошибок (403, 404, 500)

## ✅ Ключевые особенности страницы

1. **Адаптация существующей страницы** - добавление новых блоков
2. **Блок связанных скоркарт** - многие-ко-многим связь
3. **Блок артефактов** - загрузка/удаление файлов с drag & drop
4. **Блок связанных сущностей** - отображение связей через Скоркарты
5. **Навигация к связанным сущностям** - клик → детальная страница
6. **Кнопки действий по жизненному циклу** - зависят от статуса
7. **Валидация файлов** - размер и формат
8. **Progress bar** - визуализация загрузки
9. **Empty states** - для артефактов и связей
10. **Права доступа** - ПРМ (свой продукт), Методолог, Админ
11. **Responsive дизайн** - адаптация под разные разрешения
