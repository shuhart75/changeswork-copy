# Задачи для детальной страницы Скоркарты (Scorecard Detail)

## Обзор страницы

**Страница:** Детальная скоркарты (Scorecard Detail)
**Путь в макете:** Клик на скоркарту из списка → детальная страница
**Статус:** Адаптация существующей страницы

### Что показывает страница (из макете):

**Основная информация:**
- ID скоркарты (display_id)
- Название
- Текущая версия (version_number)
- Продукт
- Подпродукты (многие-ко-многим, опционально)
- Критичность (low/medium/high/critical)
- Шаблон скоркарты
- Конфигурация (JSON)
- Финансовые эффекты
- Даты (создания, обновления)
- Автор

**Новые блоки для MVP:**
- **Блок источников** - откуда создана скоркарта (Симуляция или Пилот)
- **Блок использования** - где используется скоркарта (Пилоты, Внедрения)
- **Блок артефактов** - список прикрепленных файлов

**Функционал:**
- Просмотр всех полей скоркарты
- Просмотр источников (Симуляция/Пилот)
- Просмотр использования (Пилоты/Внедрения)
- Загрузка/удаление артефактов
- Переход к связанным сущностям (клик → детальная)

**Права доступа:**
- Методолог видит все скоркарты и может редактировать
- ПРМ видит скоркарты своего продукта (только просмотр)
- Админ видит все скоркарты

## AN-SCD1: Аналитика детальной страницы Скоркарты

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 5 дней

### Summary
Детализация требований для адаптации детальной страницы Скоркарты с добавлением блоков источников, использования и артефактов

### Description

Провести аналитику для адаптации детальной страницы Скоркарты:

1. **Детализация бизнес-требований:**
   - Определить правила отображения источников
   - Уточнить логику отображения использования
   - Определить правила доступа к артефактам
   - Уточнить отображение подпродуктов

2. **Спецификация UI/UX:**
   - Дизайн блока источников (Симуляция/Пилот)
   - Дизайн блока использования (Пилоты/Внедрения)
   - Дизайн блока артефактов (список, загрузка, удаление)
   - Спецификация навигации к связанным сущностям

3. **Спецификация API контракта:**
   - GET /api/scorecards/{id}/ - детальная информация
   - GET /api/scorecards/{id}/sources/ - источники скоркарты
   - GET /api/scorecards/{id}/usage/ - использование скоркарты
   - GET /api/scorecards/{id}/artifacts/ - список артефактов
   - POST /api/scorecards/{id}/artifacts/ - загрузка артефакта
   - DELETE /api/scorecards/{id}/artifacts/{artifact_id}/ - удаление артефакта

### Acceptance Criteria

- [ ] Бизнес-требования задокументированы
- [ ] API контракт задокументирован
- [ ] UI/UX спецификация готова
- [ ] Правила доступа определены
- [ ] Документ согласован с заказчиком

### Dependencies
Нет

### Deliverables
- Документ "Спецификация детальной страницы Скоркарты" (markdown/confluence)
- API контракт для источников, использования и артефактов

         "name": "Карта с кэшбэком"

2. **API Endpoint: GET /api/scorecards/{id}/sources/**

   **Формат ответа:**

         "display_id": "SIM-001",

   **Формат ответа:**
   ```json
   {
     "pilots": [
       {
         "id": "uuid",
         "display_id": "PLT-002",
         "name": "Пилот Premium Q2 2026",

         "name": "Внедрение новой скоркарты Premium",
         "status": "deployed"
       }
     ]
   }
   ```

4. **API Endpoint: GET /api/scorecards/{id}/artifacts/**

   **Формат ответа:**
   ```json
   {
     "count": 3,
     "results": [
       {
         "id": "uuid",
         "filename": "scorecard_documentation.pdf",
         "file_size": 1536000,
         "uploaded_at": "2026-03-05T15:00:00Z",
         "uploaded_by": {
           "id": "uuid",
           "full_name": "Сидоров Сидор Сидорович"
         }
       }
     ]
   }
   ```

5. **API Endpoint: POST /api/scorecards/{id}/artifacts/**

   **Параметры запроса:**
   - `file` (multipart/form-data) - файл для загрузки

   **Бизнес-логика:**
   - Валидация: размер файла <= 50MB
   - Валидация: разрешенные форматы (pdf, xlsx, docx, txt, csv)
   - Сохранение в файловое хранилище
   - Создание записи Artifact

6. **API Endpoint: DELETE /api/scorecards/{id}/artifacts/{artifact_id}/**

   **Бизнес-логика:**
   - Валидация: пользователь имеет право удалять (автор или админ или методолог)
   - Удаление файла из хранилища
   - Удаление записи Artifact

7. **Права доступа:**
   - Просмотреть может: ПРМ (свой продукт), Методолог, Админ
   - Загрузить артефакт может: Методолог, Админ
   - Удалить артефакт может: автор артефакта, Методолог, Админ

## FE-SCD1: Frontend для детальной страницы Скоркарты

**Тип:** Story
**Приоритет:** Высокий
**Оценка:** 8 дней

### Summary
Адаптация UI детальной страницы Скоркарты с добавлением блоков источников, использования и артефактов

### Description

Адаптировать Frontend для детальной страницы Скоркарты:

1. **Компонент ScorecardDetailView:**
   - Отображение основной информации скоркарты
   - Блок источников
   - Блок использования
   - Блок артефактов
   - Responsive дизайн

2. **Основная информация:**
   - ID скоркарты (display_id)
   - Название
   - Версия
   - Продукт
   - Подпродукты (список, если указаны)
   - Критичность (бейдж с цветом)
   - Шаблон скоркарты
   - Конфигурация (JSON viewer)
   - Финансовые эффекты
   - Даты (создания, обновления)
   - Автор

3. **Блок источников:**
   - Заголовок "Источники (N)"
   - Секция "Симуляции":
     - Список симуляций-источников
     - ID, название, статус
     - Клик → переход к детальной симуляции
   - Секция "Пилоты":
     - Список пилотов-источников
     - ID, название, статус
     - Клик → переход к детальной пилота
   - Empty state если нет источников

4. **Блок использования:**
   - Заголовок "Используется в (N)"
   - Секция "Пилоты":
     - Список пилотов, использующих скоркарту
     - ID, название, статус
     - Клик → переход к детальной пилота
   - Секция "Внедрения":
     - Список изменений, использующих скоркарту
     - ID, название, статус
     - Клик → переход к детальной изменения
   - Empty state если не используется

5. **Блок артефактов:**
   - Заголовок "Артефакты (N)"
   - Список файлов:
     - Иконка типа файла
     - Название файла (кликабельное для скачивания)
     - Размер файла
     - Дата загрузки
     - Автор загрузки
     - Кнопка "Удалить" (если есть права)
   - Кнопка "Загрузить файл" (если есть права - Методолог, Админ)
   - Drag & drop зона для загрузки
   - Валидация: размер <= 50MB, форматы (pdf, xlsx, docx, txt, csv)
   - Progress bar при загрузке
   - Empty state если нет артефактов

6. **Диалог загрузки файла:**
   - Drag & drop зона
   - Кнопка "Выбрать файл"
   - Отображение выбранного файла
   - Валидация формата и размера
   - Progress bar
   - Кнопки: "Отмена", "Загрузить"

7. **Loading и Error состояния:**
   - Skeleton loader при загрузке
   - Error message при ошибке API
   - Toast уведомления при успешной загрузке/удалении

8. **API интеграция:**
   - ScorecardService.getDetail(id)
   - ScorecardService.getSources(id)
   - ScorecardService.getUsage(id)
   - ScorecardService.getArtifacts(id)
   - ScorecardService.uploadArtifact(id, file)
   - ScorecardService.deleteArtifact(id, artifactId)
   - Обработка ошибок (403, 404, 500)

## ✅ Ключевые особенности страницы

1. **Адаптация существующей страницы** - добавление новых блоков
2. **Блок источников** - откуда создана скоркарта (Симуляция/Пилот)
3. **Блок использования** - где используется скоркарта (Пилоты/Внедрения)
4. **Блок артефактов** - загрузка/удаление файлов с drag & drop
5. **Навигация к связанным сущностям** - клик → детальная страница
6. **Валидация файлов** - размер и формат
7. **Progress bar** - визуализация загрузки
8. **Empty states** - для источников, использования и артефактов
9. **Права доступа** - Методолог может загружать/удалять артефакты
10. **Responsive дизайн** - адаптация под разные разрешения
