# Ретроспективы для всех фаз MVP

## RETRO-1: Ретроспектива Фазы 1

**Тип:** Meeting
**Приоритет:** Средний
**Оценка:** 0.5 дня

### Summary
Ретроспектива после завершения Фазы 1 (Базовые списки и детальные страницы)

### Description

Провести командную ретроспективу для анализа работы в Фазе 1:

**Участники:**
- Вся команда (Аналитики, Backend, Frontend, QA)
- Product Owner / Заказчик (опционально)

**Формат:**
- Длительность: 2-3 часа
- Формат: Start-Stop-Continue или Mad-Sad-Glad

**Повестка:**

1. **Что прошло хорошо (15 мин):**
   - Успешные практики
   - Эффективные решения
   - Хорошая коммуникация

2. **Что можно улучшить (15 мин):**
   - Проблемы и блокеры
   - Неэффективные процессы
   - Технические сложности

3. **Метрики Фазы 1 (10 мин):**
   - Плановая длительность: 15 дней
   - Фактическая длительность: ?
   - Количество багов: ?
   - Velocity: ? story points
   - Отклонение от оценок: ?

4. **Технические решения (15 мин):**
   - Архитектурные решения
   - Выбор технологий
   - Code review процесс
   - Тестирование

5. **Процессы (15 мин):**
   - Планирование задач
   - Коммуникация в команде
   - Синхронизация между Backend/Frontend
   - QA процесс

6. **Action Items (30 мин):**
   - Конкретные действия для улучшения
   - Ответственные и сроки
   - Приоритизация

7. **Планирование Фазы 2 (30 мин):**
   - Обзор задач Фазы 2
   - Учет lessons learned
   - Корректировка оценок (если нужно)

### Acceptance Criteria

- [ ] Ретроспектива проведена
- [ ] Все члены команды приняли участие
- [ ] Метрики Фазы 1 собраны и проанализированы
- [ ] Action items задокументированы
- [ ] Назначены ответственные за action items
- [ ] Протокол ретроспективы сохранен

### Dependencies
- QA-PHASE1 (тестирование Фазы 1 должно быть завершено)

### Deliverables
- Протокол ретроспективы
- Список action items с ответственными
- Метрики Фазы 1

**Оценка:** 0.5 дня

## RETRO-FINAL: Финальная ретроспектива MVP

**Тип:** Meeting
**Приоритет:** Критический
**Оценка:** 1 день

### Summary
Финальная ретроспектива всего MVP проекта

### Description

Провести финальную командную ретроспективу для анализа всего проекта:

**Участники:**
- Вся команда
- Product Owner / Заказчик
- Stakeholders

**Формат:**
- Длительность: 4-6 часов
- Формат: комбинированный

**Повестка:**

1. **Обзор всего проекта (30 мин):**
   - Цели MVP
   - Что было достигнуто
   - Что не вошло в MVP

2. **Метрики проекта (30 мин):**
   - Плановая длительность: 56 дней
   - Фактическая длительность: ?
   - Общее количество багов: ?
   - Velocity по фазам
   - Отклонение от оценок по фазам
   - Критический путь: план vs факт

3. **Что прошло хорошо (45 мин):**
   - Успешные практики
   - Эффективные решения
   - Хорошая коммуникация
   - Технические достижения

4. **Что можно улучшить (45 мин):**
   - Проблемы и блокеры
   - Неэффективные процессы
   - Технические сложности
   - Коммуникационные проблемы

5. **Технические решения (1 час):**
   - Архитектура системы
   - Выбор технологий
   - Code quality
   - Тестирование
   - Performance
   - Security

6. **Процессы (1 час):**
   - Планирование
   - Оценка задач
   - Коммуникация
   - Code review
   - QA процесс
   - Deployment

7. **Lessons Learned (1 час):**
   - Что узнали о домене
   - Что узнали о технологиях
   - Что узнали о команде
   - Что узнали о процессах

8. **Рекомендации для следующих фаз (1 час):**
   - Что продолжить делать
   - Что начать делать
   - Что прекратить делать
   - Action items для post-MVP разработки

### Acceptance Criteria

- [ ] Финальная ретроспектива проведена
- [ ] Все члены команды и stakeholders приняли участие
- [ ] Метрики всего проекта собраны и проанализированы
- [ ] Lessons learned задокументированы
- [ ] Рекомендации для post-MVP разработки сформулированы
- [ ] Протокол ретроспективы сохранен
- [ ] Презентация для stakeholders подготовлена

### Dependencies
- QA-FINAL (финальное тестирование должно быть завершено)

### Deliverables
- Протокол финальной ретроспективы
- Метрики всего проекта
- Lessons learned документ
- Рекомендации для post-MVP разработки
- Презентация для stakeholders
