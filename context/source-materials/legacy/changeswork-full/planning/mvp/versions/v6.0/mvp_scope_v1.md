# MVP Scope - Система управления внедрениями риск-стратегий

**Команда:** 2 BE + 2 FE разработчика
**Срок:** 2 месяца (16 человеко-месяцев)
**Дата:** 2026-03-04

## Обязательные компоненты MVP

Следующие компоненты НЕЛЬЗЯ исключить из MVP:
- ✅ Двухэтапное согласование (approval → ratification)
- ✅ Сложные маршруты согласования (ApprovalRoute, ApprovalStage)
- ✅ Артефакты (документы)
- ✅ Полная ролевая модель (ПРМ, Методолог, Согласующий, Админ)

---

## Скоуп MVP

### 1. Базовые сущности

#### 1.1 Deployment (Внедрение)
**Функциональность:**
- Создание Deployment (название, описание, product_id)
- Просмотр списка Deployment (с фильтрацией по продукту, статусу)
- Детальный просмотр Deployment
- Автоматический расчет статуса (draft/active/deployed)
- Связь с Product (один-ко-многим)

**Упрощения:**
- Без связи с SubProduct (только Product)
- Без архивирования в MVP

#### 1.2 Simulation (Симуляция)
**Функциональность:**
- Адаптация существующего функционала
- Связь с Deployment (обязательная)
- Статусы: draft → running → completed/failed
- Базовые поля: название, описание, параметры, результаты

**Упрощения:**
- Упрощенная форма параметров (без сложных конфигураций)
- Без артефактов в MVP для Simulation

#### 1.3 Pilot (Пилот)
**Функциональность:**
- Адаптация существующего функционала
- Связь с Deployment (обязательная)
- Связь со Scorecard (многие-ко-многим, минимум 1)
- Статусы: draft → requires_activation → active → requires_correction/completed
- Базовые поля: название, описание, параметры, результаты

**Упрощения:**
- Упрощенная форма параметров
- Без артефактов в MVP для Pilot

#### 1.4 Change (Изменение)
**Функциональность:**
- Создание Change
- Связь с Deployment (обязательная)
- Связь со Scorecard (многие-ко-многим, минимум 1)
- Статусы: draft → requires_approval → approved → deployed → rolled_back
- Поля: название, описание, критичность, плановая дата внедрения
- Артефакты (прикрепление документов)

**Упрощения:**
- Без rollback функциональности в MVP (статус есть, но действие не реализовано)

#### 1.5 Scorecard (Скоркарта)
**Функциональность:**
- Создание Scorecard (название, описание, версия, конфигурация)
- Просмотр списка Scorecard
- Связь с Product
- Связь с Pilot (многие-ко-многим)
- Связь с Change (многие-ко-многим)
- Отслеживание источника (scorecard_source: simulation_id, pilot_id)

**Упрощения:**
- Без версионирования (одна версия на скоркарту)
- Без связи с SubProduct
- Упрощенная конфигурация (JSON поле без валидации)

#### 1.6 Chain (Цепочка)
**Функциональность:**
- Создание Chain (связывание Simulation → Pilot → Change)
- Просмотр Chain
- Валидация: все сущности из одного Deployment
- Поле критичности (наследуется от Change)
- Связь с Package (многие-к-одному)

**Упрощения:**
- Упрощенная цепочка: 0..1 Simulation → 0..1 Pilot → 1 Change
- Без множественных Simulation/Pilot в одной Chain

#### 1.7 Package (Пакет)
**Функциональность:**
- Создание Package (группировка Chain с одинаковым ApprovalRoute)
- Просмотр Package
- Автоматический расчет критичности (MAX от Chain)
- Отправка Package на согласование

**Упрощения:**
- Без ручного управления составом (Chain добавляются автоматически при совпадении маршрута)

### 2. Процесс согласования

#### 2.1 ApprovalRoute (Маршрут согласования)
**Функциональность:**
- Создание маршрута с двумя этапами: approval → ratification
- Связь с Product
- Список ApprovalStage (2 этапа)
- Назначение согласующих на каждый этап (ApprovalAssignment)

**Упрощения:**
- Фиксированные 2 этапа (без динамического добавления)
- Без параллельного согласования (только последовательное)

#### 2.2 Approval Process (Процесс согласования)
**Функциональность:**
- Отправка Package на согласование
- Прохождение этапов: approval → ratification
- Согласование/Отклонение с комментарием
- Уведомления в UI (без email)
- История согласований

**Упрощения:**
- Без возврата на доработку (только approve/reject)
- Без делегирования согласования

### 3. Артефакты

**Функциональность:**
- Прикрепление файлов к Change
- Прикрепление файлов к Deployment
- Просмотр списка артефактов
- Скачивание артефактов
- Удаление артефактов (только владелец)

**Упрощения:**
- Только для Change и Deployment (без Simulation/Pilot)
- Без версионирования артефактов
- Ограничение размера файла (10 MB)
- Ограничение типов файлов (PDF, DOCX, XLSX)

### 4. Ролевая модель

#### 4.1 Роли
- **ПРМ (prm)** - привязан к product_id
- **Методолог (methodologist)** - глобальная роль
- **Согласующий (approver)** - глобальная роль
- **Админ (admin)** - глобальная роль

#### 4.2 Права доступа
Полная матрица прав согласно CLAUDE.md

**Упрощения:**
- Без множественных ролей у одного пользователя в MVP
- Без динамического управления ролями (настройка через БД)

### 5. UI/UX

#### 5.1 Основные разделы
- Главная страница (дашборд с метриками)
- Deployments (список + детальный просмотр)
- Simulations (адаптация существующего)
- Pilots (адаптация существующего)
- Changes (список + детальный просмотр + форма создания)
- Scorecards (список + форма создания)
- Chains (просмотр в контексте Deployment)
- Packages (список + детальный просмотр)
- Согласования (раздел для согласующих)

#### 5.2 Адаптация под новый макет
- Единый стиль для всех форм
- Responsive дизайн
- Навигация через боковое меню
- Breadcrumbs
- Фильтрация и поиск

**Упрощения:**
- Без продвинутой аналитики и графиков
- Без экспорта данных
- Без bulk операций

---

## Что НЕ входит в MVP

- ❌ Архивирование Initiative
- ❌ Rollback Deployment (статус есть, функция нет)
- ❌ Версионирование Scorecard
- ❌ Связь с SubProduct (только Product)
- ❌ Множественные Simulation/Pilot в Chain
- ❌ Артефакты для Simulation/Pilot
- ❌ Email уведомления
- ❌ Параллельное согласование
- ❌ Делегирование согласования
- ❌ Возврат на доработку в процессе согласования
- ❌ Множественные роли у пользователя
- ❌ Динамическое управление ролями через UI
- ❌ Продвинутая аналитика
- ❌ Экспорт данных
- ❌ Bulk операции
- ❌ Audit log (детальное логирование действий)
