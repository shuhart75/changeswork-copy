# Задачи для страницы "Создание нового внедрения" (Create Deployment)

## Обзор страницы

**Страница:** Форма создания внедрения (Create Deployment Form)
**Путь в макете:** Кнопка "Создать внедрение" на странице списка внедрений
**Статус:** Новая страница, требует доработки БД и аналитики

### Что показывает страница (из макета):

**Форма создания со следующими полями:**
- Название внедрения (обязательное, текстовое поле)
- Продукт (обязательное, dropdown)
- Описание (опциональное, многострочное текстовое поле)
- Артефакты (опциональное, компонент ArtifactsManager)

**Функционал:**
- Валидация обязательных полей (название, продукт)
- Кнопка "Отмена" - возврат к списку внедрений
- Кнопка "Создать внедрение" - сохранение и переход к детальному просмотру
- Скрытие вкладок фильтров по продуктам при открытии формы

**Права доступа:**
- ПРМ может создавать внедрения только для своего продукта
- Методолог НЕ может создавать внедрения
- Админ может создавать внедрения для любого продукта

**Связь с другими сущностями:**
- Внедрение принадлежит одному Product (обязательно)
- Артефакты могут быть прикреплены сразу при создании

### Technical Notes

**Пример валидации (Django):**
```python
from django.core.exceptions import ValidationError
from django.db import models

class Deployment(models.Model):
    name = models.CharField(max_length=255)
    product = models.ForeignKey('Product', on_delete=models.PROTECT)
    
    class Meta:
        constraints = [
            models.UniqueConstraint(
                fields=['product_id', 'name'],
                name='unique_deployment_name_per_product'
            )
        ]
    
    def clean(self):
        if len(self.name) < 3:
            raise ValidationError("Название должно содержать минимум 3 символа")
```

**Автогенерация display_id:**
```python
def generate_display_id(product_id):
    last_deployment = Deployment.objects.filter(
        product_id=product_id
    ).order_by('-display_id').first()
    
    if last_deployment and last_deployment.display_id:
        last_number = int(last_deployment.display_id.split('-')[1])
        new_number = last_number + 1
    else:
        new_number = 1
    
    return f"DEP-{new_number:03d}"
```

**Права доступа (Django REST Framework):**
```python
from rest_framework import permissions

class DeploymentCreatePermission(permissions.BasePermission):
    def has_permission(self, request, view):
        user = request.user
        
        if user.role == 'methodologist':
            return False
        
        if user.role == 'admin':
            return True
        
        if user.role == 'prm':
            product_id = request.data.get('product_id')
            return product_id == user.product_id
        
        return False
```

```typescript
interface CreateDeploymentFormProps {
  onCancel: () => void;
  onSave: (deployment: Deployment) => void;
}

function CreateDeploymentForm({ onCancel, onSave }: CreateDeploymentFormProps) {
  const [deploymentName, setDeploymentName] = useState('');
  const [deploymentDescription, setDeploymentDescription] = useState('');
  const [selectedProduct, setSelectedProduct] = useState('');
  const [artifacts, setArtifacts] = useState<Artifact[]>([]);
  const [products, setProducts] = useState<Product[]>([]);
  const [loading, setLoading] = useState(false);
  const [errors, setErrors] = useState<Record<string, string>>({});

  useEffect(() => {
    fetchProducts();
  }, []);

  const handleSave = async () => {
    const newErrors: Record<string, string> = {};
    if (!deploymentName || deploymentName.length < 3) {
      newErrors.name = 'Название должно содержать минимум 3 символа';
    }
    if (!selectedProduct) {
      newErrors.product = 'Выберите продукт';
    }
    
    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors);
      return;
    }

    setLoading(true);
    try {
      const deployment = await DeploymentService.create({
        name: deploymentName,
        description: deploymentDescription,
        product_id: selectedProduct,
        artifacts: artifacts
      });
      
      showToast('Внедрение успешно создано', 'success');
      onSave(deployment);
    } catch (error) {
      if (error.status === 400) {
        setErrors(error.data);
      } else if (error.status === 403) {
        showToast('У вас нет прав для создания внедрения для этого продукта', 'error');
      } else {
        showToast('Ошибка при создании внедрения', 'error');
      }
    } finally {
      setLoading(false);
    }
  };

  return (
    <Box sx={{ width: '100%' }}>
      <Paper sx={{ p: 3, mb: 3 }}>
        <Typography variant="h6" sx={{ mb: 3 }}>
          Создание нового внедрения
        </Typography>
        <Grid container spacing={3}>
          <Grid item xs={12}>
            <TextField
              fullWidth
              label="Название внедрения"
              value={deploymentName}
              onChange={(e) => setDeploymentName(e.target.value)}
              required
              error={!!errors.name}
              helperText={errors.name}
            />
          </Grid>
          <Grid item xs={12} md={6}>
            <FormControl fullWidth required error={!!errors.product}>
              <InputLabel>Продукт</InputLabel>
              <Select
                value={selectedProduct}
                onChange={(e) => setSelectedProduct(e.target.value)}
                label="Продукт"
              >
                {products.map(product => (
                  <MenuItem key={product.id} value={product.id}>
                    {product.label}
                  </MenuItem>
                ))}
              </Select>
            </FormControl>
          </Grid>
          <Grid item xs={12}>
            <TextField
              fullWidth
              label="Описание"
              value={deploymentDescription}
              onChange={(e) => setDeploymentDescription(e.target.value)}
              multiline
              rows={4}
            />
          </Grid>
          <Grid item xs={12}>
            <ArtifactsManager artifacts={artifacts} onChange={setArtifacts} />
          </Grid>
        </Grid>
      </Paper>
      <Paper sx={{ p: 2 }}>
        <Stack direction="row" spacing={2} justifyContent="flex-end">
          <Button variant="outlined" onClick={onCancel}>
            Отмена
          </Button>
          <Button
            variant="contained"
            onClick={handleSave}
            disabled={!deploymentName || !selectedProduct || loading}
          >
            {loading ? <CircularProgress size={24} /> : 'Создать внедрение'}
          </Button>
        </Stack>
      </Paper>
    </Box>
  );
}
```

     - HTTP 400 при ошибках валидации
     - HTTP 403 при отсутствии прав
   
   - **GET /api/products/**
     - Список продуктов для ПРМ (только свой продукт)
     - Список продуктов для админа (все продукты)
     - HTTP 403 для методолога

2. **Тестирование прав доступа:**
   - **ПРМ:**
     - Может создать внедрение для своего продукта
     - Не может создать внедрение для чужого продукта (HTTP 403)
     - Видит в dropdown только свой продукт
   
   - **Методолог:**
     - Не имеет доступа к странице создания (HTTP 403)
     - Не видит кнопку "Создать внедрение" в UI
   
   - **Админ:**
     - Может создать внедрение для любого продукта
     - Видит все продукты в dropdown

3. **UI тестирование:**
   - **Отображение формы:**
     - Все поля отображаются корректно
     - Placeholder текст присутствует
     - Required маркеры на обязательных полях
     - Вкладки фильтров по продуктам скрыты
   
   - **Валидация на клиенте:**
     - Кнопка "Создать" disabled при пустом названии
     - Кнопка "Создать" disabled при невыбранном продукте
     - Error message под полем при ошибке валидации
     - Валидация длины названия (min 3, max 255)
   
   - **Динамическая загрузка:**
     - Dropdown продуктов загружается при открытии формы
     - Skeleton loader показывается при загрузке продуктов
   
   - **Кнопки действий:**
     - Кнопка "Отмена" возвращает к списку внедрений
     - Кнопка "Создать" отправляет POST запрос
     - Loading spinner на кнопке во время запроса
     - Кнопка disabled во время запроса

4. **Интеграционное тестирование:**
   - **Успешный сценарий:**
     - Заполнить форму → нажать "Создать" → получить 201 → переход к детальному просмотру
     - Toast "Внедрение успешно создано" показывается
     - Созданное внедрение появляется в списке
   
   - **Сценарий с ошибкой валидации:**
     - Заполнить форму с невалидными данными → нажать "Создать" → получить 400 → ошибки под полями
     - Исправить ошибки → повторить отправку → успех
   
   - **Сценарий с ошибкой прав:**
     - ПРМ пытается создать для чужого продукта → получить 403 → toast с сообщением об ошибке

5. **Тестирование артефактов:**
   - Создание без артефактов
   - Создание с файлами
   - Создание со ссылками
   - Валидация типов файлов
   - Валидация размера файлов (max 50MB)

6. **Responsive тестирование:**
   - Desktop (1920x1080)
   - Tablet (768x1024)
   - Mobile (375x667)
   - Все элементы доступны и кликабельны
   - Текст читаем на всех разрешениях

7. **Тестирование граничных случаев:**
   - Название ровно 3 символа (минимум)
   - Название ровно 255 символов (максимум)
   - Описание ровно 2000 символов (максимум)
   - Дублирование названия в рамках одного продукта
   - Одинаковое название для разных продуктов (должно работать)

8. **Тестирование производительности:**
   - Время загрузки формы < 1 сек
   - Время загрузки продуктов < 500ms
   - Время отправки формы < 2 сек

9. **Регрессионное тестирование:**
   - Список внедрений работает после создания нового
   - Детальный просмотр работает для созданного внедрения
   - Фильтры по продуктам работают после создания
   - Поиск находит созданное внедрение

## ✅ Ключевые особенности страницы "Создание нового внедрения"

1. **Новая страница:** Требует полной реализации (аналитика, бэкенд, фронтенд, тестирование)
2. **Доработка БД:** Constraint на уникальность name в рамках product_id, автогенерация display_id
3. **Права доступа:** ПРМ только свой продукт, методолог не может создавать, админ все продукты
4. **Валидация:** Клиентская и серверная, уникальность названия в рамках продукта
5. **Автогенерация:** display_id (DEP-XXX), status = 'draft', created_by = текущий пользователь
6. **Артефакты:** Можно прикрепить сразу при создании
7. **Скрытие фильтров:** Вкладки фильтров по продуктам скрываются при открытии формы
8. **Переход после создания:** Автоматический переход к детальному просмотру созданного внедрения
9. **Упрощение для MVP:** Без подпродуктов, только продукты
