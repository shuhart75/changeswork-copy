# Задачи для страницы "Скоркарты" (Scorecards List View)

## Обзор страницы

**Страница:** Список скоркарт (Scorecards List View)
**Путь в макете:** Боковое меню → "Скоркарты"
**Статус:** Адаптация существующей страницы

### Что показывает страница (из макете):

**Табличное представление со столбцами:**
- Дата утверждения (approved_at)
- Продукт
- Внедрение (название внедрения, к которому относится скоркарта)
- Критичность (low/medium/high/critical)
- Статус (draft/active/archived)

**Функционал:**
- Фильтрация по продукту (dropdown)
- Поиск по названию
- Сортировка по столбцам
- Пагинация
- Экспорт в Excel (с дополнительными столбцами)
- Панель инструментов:
  - Кнопка "Экспорт в Excel"
  - Кнопка "Столбцы" (выбор отображаемых столбцов)
  - Кнопка "Фильтры" (расширенные фильтры)
  - Кнопка "Высота строки"
  - Иконка "Настройки"
- Клик на строку → переход к детальному просмотру

**Права доступа:**
- ПРМ видит скоркарты своего продукта (read-only)
- Методолог видит все скоркарты и может создавать/редактировать
- Админ видит все скоркарты и может создавать/редактировать

**Особенности:**
- Скоркарты могут быть не привязаны к внедрению (показывать "—")
- Связь с внедрением через пилоты или изменения (многие-ко-многим)
- Экспорт включает дополнительные столбцы: ID, Название, Шаблон, Версия, Ручная, AR(абс), Объём выдач, NPV, RC

## BE-SC1: Backend для страницы "Скоркарты"

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 5 дней

### Summary
Разработка API endpoint для списка скоркарт с фильтрацией, пагинацией и экспортом в Excel

### Description

Разработать Backend API для страницы списка скоркарт:

1. **API Endpoint: GET /api/scorecards/**

   **Параметры запроса:**
   - `product_id` (optional) - фильтр по продукту
   - `status` (optional) - фильтр по статусу (draft/active/archived)
   - `search` (optional) - поиск по названию (case-insensitive, partial match)
   - `page` (optional, default=1) - номер страницы
   - `page_size` (optional, default=20) - размер страницы
   - `ordering` (optional, default='-created_at') - сортировка

   **Формат ответа:**
   ```json
   {
     "count": 42,
     "next": "http://api/scorecards/?page=2",
     "previous": null,
     "results": [
       {
         "id": "uuid",
         "display_id": "SC-001",
         "name": "Скоркарта Premium v2.0",
         "product": {
           "id": "uuid",
           "name": "Premium карты"
         },
         "deployment": {
           "id": "uuid",
           "display_id": "DEP-001",
           "name": "Оптимизация риск-стратегии Premium карт"
         },
         "criticality": "high",
         "status": "active",
         "approved_at": "2026-03-01T10:00:00Z",
         "created_at": "2026-02-15T10:00:00Z"
       }
     ]
   }
   ```

2. **API Endpoint: GET /api/scorecards/export/**

   **Параметры запроса:** те же, что для GET /api/scorecards/

   **Формат ответа:** Excel файл (.xlsx)

   **Столбцы в Excel:**
   - ID (SC-XXX)
   - Название
   - Продукт
   - Шаблон (template_version)
   - Версия (v1.0, v2.0)
   - Критичность
   - Статус
   - Ручная (да/нет)
   - Дата создания (DD.MM.YYYY HH:MM)
   - AR(абс) - из financial_effects
   - Объём выдач - из financial_effects
   - NPV - из financial_effects
   - RC - из financial_effects

3. **Бизнес-логика:**
   - Фильтрация по правам доступа:
     - ПРМ видит только свой продукт (read-only)
     - Методолог видит все
     - Админ видит все
   - Поиск по названию (case-insensitive, ILIKE для PostgreSQL)
   - Сортировка по умолчанию: -created_at (новые первые)
   - Отображение внедрения через связи с пилотами/изменениями (многие-ко-многим)
   - Если скоркарта не привязана к внедрению → deployment = null

4. **Права доступа:**
   - Middleware проверяет роль пользователя
   - Автоматическая фильтрация по product_id для ПРМ
   - HTTP 403 если ПРМ пытается получить чужой продукт

5. **Оптимизация:**
   - Select related для product, deployment
   - Prefetch related для pilots, changes (для определения deployment)
   - Индексы на product_id, status, created_at

6. **Экспорт в Excel:**
   - Использовать библиотеку openpyxl или xlsxwriter
   - Применять те же фильтры, что и для списка
   - Заголовки на русском языке
   - Форматирование дат: DD.MM.YYYY HH:MM
   - Content-Disposition: attachment; filename="scorecards_export_YYYY-MM-DD.xlsx"

### Technical Notes

**Пример определения внедрения через связи:**
```python
def get_deployment(scorecard):
    """Получить внедрение через связи с пилотами/изменениями"""
    # Через пилоты
    pilots = scorecard.pilots.all()
    for pilot in pilots:
        if pilot.deployment:
            return pilot.deployment

    # Через изменения
    changes = scorecard.changes.all()
    for change in changes:
        if change.deployment:
            return change.deployment

    return None
```

**Пример SQL оптимизации (Django ORM):**
```python
from django.db.models import Q

Scorecard.objects.filter(
    Q(name__icontains=search),
    product_id=product_id,
    status=status
).select_related(
    'product'
).prefetch_related(
    'pilots__deployment',
    'changes__deployment'
).order_by('-created_at')
```

## FE-SC1: Frontend для страницы "Скоркарты"

Разработка UI страницы списка скоркарт с таблицей, фильтрами, панелью инструментов и экспортом в Excel

### Description

Разработать Frontend для страницы списка скоркарт:

1. **Компонент ScorecardsView:**
   - Табличное представление (Material-UI Table)
   - Столбцы: Дата утверждения, Продукт, Внедрение, Критичность, Статус
   - Responsive дизайн

2. **Панель инструментов (Toolbar):**
   - Кнопка "Экспорт в Excel" (скачивание файла)
   - Кнопка "Столбцы" (выбор отображаемых столбцов)
   - Кнопка "Фильтры" (расширенные фильтры)

   - Dropdown фильтр по продукту
   - Dropdown фильтр по статусу
   - Поле поиска по названию (debounce 300ms)

4. **Сортировка:**

   - Выбор размера страницы (10/20/50)

6. **Статусные бейджи:**
   - draft: серый
   - active: зеленый
   - archived: серый с opacity

7. **Критичность:**
   - Цветной чип:
     - low: зеленый
     - medium: желтый
     - high: оранжевый
     - critical: красный

8. **Отображение внедрения:**
   - Если скоркарта привязана к внедрению → название внедрения (кликабельное)
   - Если не привязана → "—"

9. **Экспорт в Excel:**

   - Применение текущих фильтров к экспорту
   - Скачивание файла через браузер
   - Toast уведомление "Экспорт начат..."

10. **Навигация:**
    - Клик на строку → переход к детальному просмотру
    - Breadcrumbs: "Скоркарты"

11. **Loading и Error состояния:**
    - Skeleton loader при загрузке
    - Error message при ошибке API
    - Empty state если нет данных

12. **API интеграция:**
    - ScorecardService.getList(filters, page, pageSize, ordering)
    - ScorecardService.exportToExcel(filters)
    - Обработка ошибок (403, 404, 500)
    - Toast уведомления при ошибках
