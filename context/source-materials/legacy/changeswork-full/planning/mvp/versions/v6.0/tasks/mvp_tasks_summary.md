## Соответствие доменной модели

### ✅ Реализовано:
- Жизненные циклы: Simulation, Pilot, Change
- Процесс согласования (двухэтапный)
- Интеграция ЖЦ с процессом согласования
- Связи через Скоркарты (многие-ко-многим)
- Артефакты для всех сущностей
- Отслеживание источников скоркарт (таблица scorecard_source)
- Автоматический расчет критичности Change
- **Визуализация lineage** - через блоки "Связанные сущности" на детальных страницах
- **Отслеживание цепочки** - Simulations → Scorecards → Pilots → Scorecards → Changes

### 📝 Примечание:
Chain как отдельная сущность была удалена из модели. Связи отслеживаются через:
- Таблицу `scorecard_source` (откуда создана скоркарта: Simulation или Pilot)
- Связи многие-ко-многим между Pilot-Scorecard и Change-Scorecard
- Блоки "Связанные сущности" на детальных страницах показывают всю цепочку

При создании новых Pilots/Changes связи автоматически устанавливаются через выбранные Скоркарты.
