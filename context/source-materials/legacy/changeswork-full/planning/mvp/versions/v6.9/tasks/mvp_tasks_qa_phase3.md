# QA задачи для Фазы 3: Backend процессы

## QA-PHASE3: Тестирование Фазы 3

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 3 дня

### Summary
Интеграционное тестирование жизненных циклов и процесса согласования

### Description

Провести полное тестирование backend процессов Фазы 3:

**Компоненты для тестирования:**
1. Жизненный цикл изменения (BE-CL1)
2. Процесс согласования/утверждения (BE-AP1)
3. Интеграция ЖЦ с процессом (BE-INT1)
4. Жизненный цикл пилота (BE-PL1)
5. Система уведомлений (BE-NOT1)
6. Жизненный цикл симуляции (BE-SL1)

**Тестовые сценарии:**

**1. ЖЦ симуляции (0.5 дня):**
- Создание симуляции (статус = draft)
- Переход draft → running
- Переход running → completed
- Переход running → failed
- Валидация: нельзя перейти из completed/failed в другие статусы
- Независимость от процесса согласования

**2. ЖЦ изменения (0.5 дня):**
- Создание изменения (статус = draft)
- Валидация: минимум 1 скоркарта обязательна
- Автоматический расчет критичности = MAX(критичности всех скоркарт)
- Переход draft → requires_approval
- Переход requires_approval → approved (через процесс согласования)
- Переход approved → deployed
- Переход deployed → rolled_back (с указанием причины)
- Валидация: только одно активное внедрение на изменение (rollback_at = null)

**3. ЖЦ пилота (0.5 дня):**
- Создание пилота (статус = draft)
- Валидация: минимум 1 скоркарта обязательна
- Переход draft → requires_activation
- Переход requires_activation → active (через процесс согласования)
- Повторная отправка пилота на согласование/утверждение после изменений (через `submit_for_approval`, переходы в рамках процесса согласования)
- Переход active → completed

**4. Процесс согласования (1 день):**
- Двухэтапный процесс: Approval → Ratification
- Отправка пилота на согласование (status = requires_activation):
  - Выбор маршрута согласования
  - Назначение согласующих для этапа Approval
  - Назначение утверждающих для этапа Ratification
  - Создание Package (если несколько пилотов с одинаковым маршрутом)
- Отправка изменения на согласование (status = requires_approval):
  - Аналогично пилоту
- Согласование (Approval):
  - Согласующий видит свои назначения
  - Кнопки: "Согласовать", "Отклонить"
  - При согласовании: переход к этапу Ratification
  - При отклонении: возврат в draft (для пилота) или requires_approval (для изменения)
  - Комментарий опционален при отклонении
- Утверждение (Ratification):
  - Утверждающий видит свои назначения
  - Кнопки: "Утвердить", "Отклонить"
  - При утверждении: переход в approved (для изменения) или active (для пилота)
  - При отклонении: возврат в draft
  - Комментарий опционален при отклонении
- Batch операции:
  - Массовое согласование/отклонение из Package
  - Все пилоты/изменения в Package обрабатываются одновременно

**5. Система уведомлений (0.5 дня):**
- Email уведомления отправляются при:
  - Отправке на согласование (согласующим)
  - Согласовании (утверждающим)
  - Утверждении (автору)
  - Отклонении (автору)
  - Переходе в deployed (всем участникам)
  - Rollback (всем участникам)
- Шаблоны email с брендингом
- Корректность данных в письмах
- История уведомлений сохраняется

**6. Интеграционное тестирование (0.5 дня):**
- Полный цикл: Создание изменения → Отправка на согласование → Согласование → Утверждение → Deployed → Rollback
- Полный цикл: Создание пилота → Отправка на активацию → Согласование → Утверждение → Active → Completed
- Создание Package с несколькими изменениями → Batch согласование
- Проверка всех уведомлений на каждом этапе

- [ ] ЖЦ изменения: валидация минимум 1 скоркарты работает
- [ ] ЖЦ пилота: все переходы работают корректно
- [ ] ЖЦ пилота: валидация минимум 1 скоркарты работает
- [ ] Процесс согласования: двухэтапный процесс работает
- [ ] Процесс согласования: назначение согласующих/утверждающих работает
- [ ] Процесс согласования: согласование/отклонение работает
- [ ] Процесс согласования: комментарии при отклонении обязательны
- [ ] Процесс согласования: batch операции работают
- [ ] Интеграция ЖЦ с процессом: автоматические переходы работают
- [ ] Система уведомлений: все типы уведомлений отправляются
- [ ] Система уведомлений: шаблоны корректны
- [ ] Система уведомлений: история сохраняется
- [ ] Права доступа работают корректно
- [ ] Полные интеграционные сценарии проходят успешно
- [ ] Все найденные баги задокументированы
- [ ] Критические баги исправлены

- BE-PL1 (ЖЦ пилота)
- BE-NOT1 (уведомления)
- BE-SL1 (ЖЦ симуляции)
