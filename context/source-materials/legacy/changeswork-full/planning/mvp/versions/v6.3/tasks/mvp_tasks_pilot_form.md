# Задачи для формы создания/редактирования Пилота

## Обзор

**Компонент:** Форма создания/редактирования Пилота
**Путь в макете:** Кнопка "Создать пилот" → форма / Кнопка "Редактировать" на детальной → форма
**Статус:** Адаптация существующей формы

### Что включает форма:

**Основные поля:**
- Название (обязательное)
- Внедрение (выбор из списка, обязательное)
- Скоркарты (множественный выбор, обязательно минимум 1)
- Описание (опционально)

**Функционал:**
- Создание нового пилота
- Редактирование существующего пилота
- Валидация полей
- Сохранение в статусе draft

**Права доступа:**
- Создать может: ПРМ (свой продукт), Админ
- Редактировать может: ПРМ (свой продукт), Админ

## BE-PF1: Backend для формы Пилота

**Тип:** Story
**Приоритет:** Средний
**Оценка:** 3 дня

### Summary
Адаптация Backend для создания/редактирования Пилота

### Description

Адаптировать Backend для формы Пилота:

1. **API Endpoint: POST /api/pilots/**

   **Параметры запроса:**
   ```json
   {
     "name": "Пилот Premium Q1 2026",
     "deployment_id": "uuid",
     "scorecard_ids": ["uuid1", "uuid2"],
     "description": "Описание пилота"
   }
   ```

   **Бизнес-логика:**
   - Валидация: название обязательно
   - Валидация: deployment_id обязателен и существует
   - Валидация: scorecard_ids обязателен, минимум 1 скоркарта
   - Валидация: все скоркарты существуют и принадлежат тому же продукту
   - Валидация: пользователь имеет право создавать (ПРМ своего продукта, Админ)
   - Создание пилота в статусе draft
   - Установка created_by = текущий пользователь
   - Связывание со скоркартами (многие-ко-многим)

2. **API Endpoint: PUT /api/pilots/{id}/**

   **Параметры запроса:**
   ```json
   {
     "name": "Обновленное название",
     "scorecard_ids": ["uuid1", "uuid2", "uuid3"],
     "description": "Обновленное описание"
   }
   ```

   **Бизнес-логика:**
   - Валидация: пользователь имеет право редактировать
   - Валидация: пилот в статусе draft (нельзя редактировать active/completed)
   - Валидация: минимум 1 скоркарта
   - Обновление полей
   - Обновление связей со скоркартами

3. **Права доступа:**
   - Создать может: ПРМ (свой продукт), Админ
   - Редактировать может: ПРМ (свой продукт), Админ
   - HTTP 403 если нет прав

### Acceptance Criteria

- [ ] POST /api/pilots/ реализован
- [ ] PUT /api/pilots/{id}/ реализован
- [ ] Валидация полей работает
- [ ] Валидация минимум 1 скоркарты работает
- [ ] Связывание со скоркартами работает
- [ ] Права доступа реализованы
- [ ] Unit тесты покрывают все сценарии (coverage > 80%)

## FE-PF1: Frontend для формы Пилота

**Тип:** Story
**Приоритет:** Средний
**Оценка:** 3 дня

### Summary
Адаптация UI формы создания/редактирования Пилота

### Description

Адаптировать Frontend для формы Пилота:

1. **Компонент PilotForm:**
   - Режим создания (create)
   - Режим редактирования (edit)
   - Валидация полей
   - Responsive дизайн

2. **Поля формы:**
   - Название (TextField, обязательное, max 255 символов)
   - Внедрение (Autocomplete, обязательное, только для создания)
   - Скоркарты (Autocomplete multiple, обязательное, минимум 1)
   - Описание (TextField multiline, опционально)

3. **Валидация:**
   - Название: обязательное, не пустое
   - Внедрение: обязательное (только при создании)
   - Скоркарты: обязательное, минимум 1 выбрана
   - Показ ошибок под полями

4. **Autocomplete для скоркарт:**
   - Множественный выбор
   - Поиск по названию
   - Отображение: ID + название + критичность
   - Фильтрация по продукту внедрения

5. **Кнопки:**
   - "Отмена" - закрытие формы без сохранения
   - "Сохранить" - сохранение и переход к детальной

6. **Loading и Error состояния:**
   - Disabled кнопка "Сохранить" при отправке
   - Spinner при сохранении
   - Toast уведомления при успехе/ошибке

7. **API интеграция:**
   - PilotService.create(data)
   - PilotService.update(id, data)
   - DeploymentService.getList() - для выбора внедрения
   - ScorecardService.getList(productId) - для выбора скоркарт
   - Обработка ошибок (400, 403, 500)

### Acceptance Criteria

- [ ] PilotForm компонент реализован
- [ ] Режим создания работает
- [ ] Режим редактирования работает
- [ ] Валидация полей работает
- [ ] Валидация минимум 1 скоркарты работает
- [ ] Autocomplete для внедрения работает
- [ ] Autocomplete для скоркарт работает (множественный выбор)
- [ ] Фильтрация скоркарт по продукту работает
- [ ] Кнопки "Отмена" и "Сохранить" работают
- [ ] Loading состояние работает
- [ ] Error состояние отображается
- [ ] Toast уведомления работают
- [ ] Responsive дизайн работает
- [ ] API интеграция работает корректно
- [ ] Unit тесты для компонента (coverage > 70%)

### Dependencies
