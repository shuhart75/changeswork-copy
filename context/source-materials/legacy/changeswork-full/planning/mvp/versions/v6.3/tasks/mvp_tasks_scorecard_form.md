# Задачи для формы создания/редактирования Скоркарты

## Обзор

**Компонент:** Форма создания/редактирования Скоркарты
**Путь в макете:** Кнопка "Создать скоркарту" → форма / Кнопка "Редактировать" на детальной → форма
**Статус:** Новая разработка

### Что включает форма:

**Основные поля:**
- Название (обязательное)
- Продукт (выбор из списка, обязательное)
- Подпродукты (множественный выбор, опционально)
- Критичность (выбор: low/medium/high/critical, обязательное)
- Версия скоркарты (read-only, создаётся системой как целое число при сохранении)
- Версия шаблона скоркарты (выбор из списка, обязательное)
- Конфигурация (JSON editor, обязательное)
- Финансовые эффекты (JSON editor, опционально)
- Описание (опционально)

**Функционал:**
- Создание новой скоркарты
- Редактирование существующей скоркарты
- Валидация полей
- JSON editor для конфигурации
- Загрузка конфигурации из шаблона
 - **Версионирование:** любое изменение создаёт новую `ScorecardVersion` (вместо in-place update)

**Права доступа:**
- Создать может: Методолог, Админ
- Редактировать может: Методолог, Админ

## BE-SCF1: Backend для формы Скоркарты

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 4 дня

### Summary
Разработка Backend для создания/редактирования Скоркарты

### Description

Разработать Backend для формы Скоркарты:

1. **API Endpoint: POST /api/scorecards/**

   **Параметры запроса:**
   ```json
   {
     "name": "Скоркарта Premium v2.0",
     "product_id": "uuid",
     "subproduct_ids": ["uuid1", "uuid2"],
     "criticality": "high",
     "template_version_id": "uuid",
     "configuration": {...},
     "financial_effects": {...},
     "description": "Описание скоркарты"
   }
   ```

   **Бизнес-логика:**
   - Валидация: название обязательно
   - Валидация: product_id обязателен и существует
   - Валидация: subproduct_ids опциональны, если указаны - должны принадлежать product
   - Валидация: criticality обязателен, один из (low/medium/high/critical)
   - Валидация: template_version_id обязателен и существует
   - Валидация: configuration обязателен, валидный JSON
   - Валидация: пользователь имеет право создавать (Методолог, Админ)
   - Создание `Scorecard` + первой `ScorecardVersion` (version_number = 1)
   - Установка created_by = текущий пользователь
   - Связывание с подпродуктами (многие-ко-многим)

2. **API Endpoint: PUT /api/scorecards/{id}/**

   **Параметры запроса:**
   ```json
   {
     "name": "Обновленное название",
     "subproduct_ids": ["uuid1", "uuid2", "uuid3"],
     "criticality": "critical",
     "configuration": {...},
     "financial_effects": {...},
     "description": "Обновленное описание"
   }
   ```

   **Бизнес-логика:**
   - Валидация: пользователь имеет право редактировать (Методолог, Админ)
   - Обновление метаданных `Scorecard` (name/criticality/description)
   - **Создание новой `ScorecardVersion`** (version_number = prev + 1) с `configuration/financial_effects`
   - Обновление связей с подпродуктами

3. **API Endpoint: GET /api/scorecard-template-versions/{id}/default-config/**

   **Формат ответа:**
   ```json
   {
     "configuration": {
       "threshold": 0.5,
       "weights": {...},
       "features": [...]
     }
   }
   ```

   **Бизнес-логика:**
   - Получение конфигурации по умолчанию из версии шаблона
   - Используется для предзаполнения формы

4. **Права доступа:**
   - Создать может: Методолог, Админ
   - Редактировать может: Методолог, Админ
   - HTTP 403 если нет прав

### Acceptance Criteria

- [ ] POST /api/scorecards/ реализован

## FE-SCF1: Frontend для формы Скоркарты

**Тип:** Story
**Приоритет:** Критический
**Оценка:** 10 дней

### Summary
Разработка UI формы создания/редактирования Скоркарты с JSON editor

### Description

Разработать Frontend для формы Скоркарты:

1. **Компонент ScorecardForm:**
   - Режим создания (create)
   - Режим редактирования (edit)
   - Валидация полей
   - JSON editor для конфигурации
   - Responsive дизайн

2. **Поля формы:**
   - Название (TextField, обязательное, max 255 символов)
   - Версия скоркарты (read-only, отображение текущего `version_number`)
   - Продукт (Autocomplete, обязательное, только для создания)
   - Подпродукты (Autocomplete multiple, опционально)
   - Критичность (Select, обязательное: low/medium/high/critical)
   - Версия шаблона скоркарты (Autocomplete, обязательное)
   - Конфигурация (JSON editor, обязательное)
   - Финансовые эффекты (JSON editor, опционально)
   - Описание (TextField multiline, опционально)

3. **Валидация:**
   - Название: обязательное, не пустое
   - Продукт: обязательное (только при создании)
   - Критичность: обязательное
   - Шаблон: обязательное
   - Конфигурация: обязательное, валидный JSON
   - Показ ошибок под полями

4. **JSON Editor:**
   - Syntax highlighting
   - Валидация JSON в реальном времени
   - Показ ошибок парсинга
   - Кнопка "Загрузить из шаблона" (при выборе шаблона)
   - Форматирование (prettify)

5. **Autocomplete для подпродуктов:**
   - Множественный выбор
   - Фильтрация по выбранному продукту
   - Если не выбраны - скоркарта относится ко всем подпродуктам

6. **Загрузка конфигурации из шаблона:**
   - При выборе шаблона - кнопка "Загрузить конфигурацию"
   - Загружает default configuration из шаблона
   - Предупреждение если конфигурация уже заполнена

7. **Кнопки:**
   - "Отмена" - закрытие формы без сохранения
   - "Сохранить" - сохранение и переход к детальной

8. **Loading и Error состояния:**
   - Disabled кнопка "Сохранить" при отправке
   - Spinner при сохранении
   - Toast уведомления при успехе/ошибке

9. **API интеграция:**
   - ScorecardService.create(data)
   - ScorecardService.update(id, data)
   - ProductService.getList() - для выбора продукта
   - SubProductService.getList(productId) - для выбора подпродуктов
   - ScorecardTemplateService.getList() - для выбора шаблона
   - ScorecardTemplateService.getDefaultConfig(templateId) - для загрузки конфигурации
   - Обработка ошибок (400, 403, 500)
