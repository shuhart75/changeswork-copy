# Command Cheatsheet

Короткая шпаргалка по естественным командам для LLM. Это не shell-команды и не slash-команды. Пиши их обычным сообщением в чат, а после первой строки добавляй контекст: пути, feature, slice, task ids, даты, expected output.

## Базовые переключатели режима

1. `новая фича`
2. `занимаемся планированием`
3. `делаем требования`
4. `делаем презентационный прототип`
5. `делаем прототип для разработки`
6. `обновляем прогресс`
7. `финализируем релиз`

## Готовые фразы для ежедневной работы

### Планирование

- `новая фича, в папке лежит внешний пакет материалов, сначала сделай intake и не создавай feature раньше времени`
- `новая фича, нужно отделить existing baseline coverage от реальной новой дельты`
- `занимаемся планированием, нужно разложить доработку на planning stories и дать HLE в человеко-днях`
- `занимаемся планированием, собери quarter-plan и commander-plan по текущему scope`
- `разложи фичу на planning stories, отдельно покажи ideal scope и MVP`
- `подготовь HLE, я хочу якорные оценки по stories для обсуждения с командой`
- `сравни план и факт по фиче, покажи где уже выходим за commander plan`

### Требования

- `делаем требования, в папке лежат скрины и change request, нужно собрать requirement pack`
- `делаем требования, разложи фичу по смысловым slices и подготовь FE/BE packs`
- `актуализируй требования по feature, проверь влияние на соседние фичи и baseline`
- `зафиксируй доменное решение и обнови domain-impact`
- `проверь консистентность требований по feature и выпиши что нужно дотянуть`

### Прототипы

- `делаем презентационный прототип, нужен кликабельный макет для согласования scope`
- `покажи ideal и MVP на презентационном прототипе`
- `делаем прототип для разработки, нужен handoff-макет под конкретный slice`
- `актуализируй прототипы по backlog`
- `актуализируй прототип по требованиям и перечисли что изменилось`

### Прогресс и задачи

- `обновляем прогресс, задача RSCON-2445 завершена вчера`
- `обновляем прогресс, задачу RSCON-2451 взял второй фронтендер позавчера`
- `добавь реальные задачи вместо story STORY-DEPLOYMENTS-003`
- `обнови реальный прогресс и пересобери actual-progress gantt`
- `добавь milestone релиза на 2026-04-30`

### Релиз и baseline

- `финализируем релиз, собери релизный пакет по вовлечённым фичам`
- `зафиксируй итоговые требования по релизу`
- `закрой consistency backlog по релизу или явно пометь отложенное`
- `промоуть в baseline релиз releases/2026-Q2/rscon-2438`
- `откати решение DEC-2026-04-23-SCORECARDS-001`

## Рекомендуемый шаблон сообщения

```text
новая фича
Источник: `/home/reutov/Documents/AI/simulations_AI_agent`
Квартал: `2026-Q2`
Нужно сделать intake: сопоставить с baseline/current, existing features и legacy planning, ничего не создавать до отдельного подтверждения.
```

```text
делаем требования
В папке `context/source-materials/change-requests/packages-filtering` лежат скрины и описание.
Feature: `packages`
Slice: `filtering`
Нужно подготовить FE/BE requirement packs по шаблону, обновить domain-impact и перечислить затронутые прототипы.
```

## Короткие правила

- первая строка — команда;
- дальше — конкретный контекст;
- если меняется режим работы, начинай с базового переключателя режима;
- если делаешь точечное действие внутри текущего режима, используй команду второго уровня;
- если изменение влияет на домен или соседние фичи, в том же сообщении проси обновить `domain-impact.md` и `consistency-backlog.md`.

## Полезные синонимы

- Вместо `делаем требования`: `пишем требования`, `соберем требования`, `давай сделаем требования`
- Вместо `занимаемся планированием`: `давай планировать`, `переходим в планирование`, `пора планировать`
- Вместо `делаем презентационный прототип`: `нужен демо-макет`, `собери кликабельный макет для заказчика`
- Вместо `делаем прототип для разработки`: `собери handoff-макет`, `нужен макет для фронтендера`
- Вместо `обновляем прогресс`: `обновим статус задач`, `зафиксируем факт`, `обнови фактический прогресс`
- Вместо `финализируем релиз`: `готовим релизный пакет`, `закрываем релизный цикл`

- Вместо `новая фича`: `разбери новую фичу`, `сделай feature intake`, `это новая фича`
