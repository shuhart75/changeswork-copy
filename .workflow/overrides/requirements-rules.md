# Requirements Rules Override

- Канонические шаблоны requirements лежат в `.workflow/templates/requirements/`.
- Для требований доступны два формата: новый лёгкий `*.readable.template.md` и старый подробный `*.template.md`.
- Если пользователь явно указал формат, использовать его; если формат не указан, существующая фича сохраняет текущий формат, новая фича по умолчанию получает новый лёгкий формат.
- Backend-требования должны включать OpenAPI-фрагмент и примеры запросов/ответов.
- Описание модели данных оформляется markdown-таблицей.
- Requirement packs живые и могут дополняться в ходе разработки до релизной фиксации.
