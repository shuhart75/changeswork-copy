# Context — Identity and Access

## Purpose
Управление пользователями, ролями и правилами доступа к доменным сущностям и процессу согласования.

## Main objects
- `User`
- `UserRole`

## Key rules
- У пользователя может быть несколько ролей.
- `prm` и `methodologist` привязаны к продукту (`product_id` обязателен).
- `approver`, `ratifier`, `admin` — глобальные роли.
- ПРМ и методолог могут дополнительно быть approver/ratifier.
- Комбинация `prm + methodologist` запрещена.
- `Approver` и `Ratifier` видят только свои назначения в разделе согласований.
- Для страницы `Пакеты` действует отдельное доменное исключение: `methodologist` может собирать пакет из элементов разных продуктов.
