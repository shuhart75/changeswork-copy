# Context — Scorecards

## Purpose
Управление скоркартами как самостоятельным доменным агрегатом, используемым версиями пилотов и внедрений.

## Main objects
- `Scorecard`
- `ScorecardVersion`
- `ScorecardSource`

## Key rules
- Скоркарта создаётся только из контекста `Pilot` или `Deployment`.
- Может быть ручной или частично предзаполненной по `Simulation`.
- Каждое содержательное изменение создаёт новую `ScorecardVersion`.
- Критичность считается на фронте по порогам шаблона и сохраняется в версии скоркарты.
- Скоркарта не имеет собственного `ApprovalInstance`.
- Статус `Scorecard` зависит только от наличия связей использования с `PilotVersion`/`DeploymentVersion`.
- Если скоркарта закреплена как обязательная lineage-скоркарта у `simulation_based` внедрения, удалить её связь с этим внедрением нельзя.
